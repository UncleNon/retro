# 11. 進行計画 / 体制 / QA / リスク管理
| **段階** | **期間目安** | **成果物**                 | **ゲート条件**                                      |
|----------|--------------|----------------------------|-----------------------------------------------------|
| 0        | 2週間        | 調査凍結、スタイルバイブル | 画面密度、パレット、世界観を固定。                  |
| 1        | 8週間        | 垂直スライス               | 町1、ルート1、ダンジョン1、18体、技能18種で遊べる。 |
| 2        | 8週間        | 基幹システム完成           | 契約、合成、図鑑、セーブ、ショップが動く。          |
| 3        | 16週間       | 本制作                     | 全地域、主要ストーリー、主要アセット実装。          |
| 4        | 6週間        | アルファ                   | 最後まで通しで遊べる。重大バグ修正フェーズ。        |
| 5        | 6週間        | ベータ/磨き込み            | QoL、バランス、移植、審査準備。                     |

| **リスク**         | **発生要因**                         | **影響**           | **対策**                                |
|--------------------|--------------------------------------|--------------------|-----------------------------------------|
| スコープ肥大       | モンスター数とマップ数を同時に増やす | 終わらない         | vertical slice通過前は数を増やさない    |
| ドットの不統一     | AI出力をそのまま混ぜる               | 全体の質が崩れる   | スタイルバイブル固定 + Aseprite最終化   |
| 合成表の爆発       | 総当たり配合を狙う                   | 管理不能           | rule-based を主軸に exact recipe を限定 |
| モバイル可読性不足 | 文字とボタンが小さい                 | 遊べない           | 320x288基準 + 実機検証                  |
| 法務リスク         | 既存IPに寄せ過ぎる                   | 販売不可・差し替え | オリジナル命名・形状・UI・地形          |
| 終盤のバランス崩壊 | 希少配合だけが強過ぎる               | 単調化             | 特性と役割分散でメタを散らす            |

> **QAの最小完了条件**
>
> A) 章1〜8を通してソフトロックがない。B) 合成・預かり・セーブ/ロードでデータ破損がない。C) モバイルで30分連続プレイしても操作が破綻しない。D) 代表的なタイル継ぎ目、UI拡大、状態異常重複、特性継承、章進行分岐を自動/手動テストで再現できる。
