# 付録B. プロンプト雛形
> **B-1 スタイルバイブル作成用（初回）**
>
> Create an original handheld-era monster-collection RPG art bible.
> Show: 1) master palette, 2) grass/dirt/cliff tiles, 3) one town NPC,
> 4) one monster icon, 5) one battle monster sprite, 6) one UI window.
> Constraints: original artwork only. No logos. No watermarks.
> Hard-edged pixel art. No anti-aliasing. Max 16 colors per asset.
> Top-down orthographic for map assets. Clean pixel clusters.
> Transparent background for asset samples.

> **B-2 主要キャラ参照シート（Nano Banana 2 / Pro 向け）**
>
> Use the uploaded reference as the canonical design for character
> 'Rion'. Create 4 consistent views: front, 3/4, side, battle pose.
> Keep body ratio, hair silhouette, scarf length, boots, and palette stable.
> Style: authentic 16-bit-inspired pixel art, crisp clusters, stepped shading,
> no blur, no painterly textures, no text.

> **B-3 差分編集（GPT Image 1.5 向け）**
>
> Use the same character from the input image. Preserve silhouette,
> head-to-body ratio, line weight, palette family, and costume structure.
> Change ONLY the scarf accent from blue to rust red and add a small satchel.
> Do not redesign the character. No text. No logos. No watermarks.

> **B-4 タイルセット案出し**
>
> Create a seamless top-down pixel art tileset for a forest route.
> Include grass, dirt path, cliff edge, tree base, flowers, signpost, fence,
> small pond edge, and shadow variants. Tile logic must be readable.
> 16x16 tile intent. Hard edges only. No anti-aliasing. Transparent background.

> **B-5 戦闘モンスター案出し**
>
> Create an original monster battle sprite for a rank-D spirit family creature.
> Readable silhouette, one clear focal feature, limited palette,
> front-facing battle pose, transparent background, no props,
> no text, original IP only.
