# 8. アート / 音 / 素材の制作要件
Asepriteはtilemap layer、linked cels、CLI exportを提供しており [R7]、最終アセット量産に向く。ただし“低解像度だから少ない工数”とは考えないこと。携帯機級の品質再現は、1px単位の輪郭整理と、タイル文法の一貫性に大量の手戻りが発生する。したがって、アセットは数を先に積み、各カテゴリの完成条件を固定する。

| **カテゴリ**              | **数量目安** | **基準サイズ**      | **備考**                                       |
|---------------------------|--------------|---------------------|------------------------------------------------|
| 地形タイルセット          | 22 sheet     | 16x16基準           | 地表、建物、洞窟、神殿、雪、港、屋内。         |
| 大型背景パーツ            | 80+          | 32x32〜64x64        | 塔、木、船、橋、祭壇。                         |
| 主人公/主要NPCシート      | 18           | 16x24, 4方向x3frame | 主人公、ライバル、幹部、店主など。             |
| 一般NPCシート             | 24           | 16x24, 4方向x3frame | 町民、旅人、漁師、研究員。                     |
| モンスター戦闘前面        | 120          | 48x48〜64x64        | 敵表示用。大型種は64x64許可。                  |
| モンスターアイコン/図鑑顔 | 120          | 16x16 / 24x24       | メニュー、図鑑、編成。                         |
| 技能エフェクト            | 32           | 16x16〜64x64        | 再着色前提で共用率を高める。                   |
| UIアトラス                | 6            | 可変                | 枠、カーソル、状態異常、属性、ボタン。         |
| BGM                       | 18 track     | —                   | 町、ルート、各ダンジョン、ボス、裏迷宮、勝利。 |

| **規則**         | **基準**     | **意図**                                            |
|------------------|--------------|-----------------------------------------------------|
| マスターパレット | 48色前後     | 全体の統一感を作る。biomeごとに部分パレットを切る。 |
| 1アセット内色数  | 原則12〜16色 | 輪郭と塊を優先し、にじみを防ぐ。                    |
| アンチエイリアス | 禁止         | 出荷ドットは硬いピクセル境界を守る。                |
| 影方向           | 左上光源固定 | 地形・キャラ・UIの世界観を揃える。                  |
| 歩行アニメ       | 3frame       | 古典携帯機のテンポに寄せる。                        |
| 待機アニメ       | 1〜2frame    | 落ち着いた画面を保ちつつ、生気だけ持たせる。        |

> **重要: AI出力をそのまま最終ドットにしない**
>
> AIは『雰囲気の近いラフ』『複数案』『構図探索』『衣装差分』『UIモック』には使えるが、タイルの継ぎ目、1px輪郭、歩行アニメ、色数管理、パレット固定、衝突形状整合は人手で詰める必要がある。特にタイルセットとUIは、最終版を必ずAsepriteで再整形する。
