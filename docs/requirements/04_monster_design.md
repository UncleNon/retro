## 4.2 モンスターデザイン哲学

### 基本理念
> 「400体全てに存在理由がある。一体たりとも使い回しや水増しを許さない。」

### 400体で足りると判断する理由
- 400体はソロ開発のモンスター育成RPGとしては十分に大規模で、配合・育成・図鑑収集の軸を成立させられる
- 問題は素の体数より、各体にどれだけ役割差・系譜差・入手機会差を持たせられるか
- 400を超えても、見た目だけ違う水増しが増えると価値が下がる
- 逆に 400体でも、配合、個体差、変異種、出現世界差、スキル差があれば体感量はもっと大きくできる

### 体感量を増やす設計
- **変異種**: 色違い、形態差、覚醒で同一種の掘り下げを作る
- **地域差**: 同系統でも出現世界に応じて属性や挙動、図鑑文の解釈を変える
- **個体差**: 性格、忠誠度、継承スキルで同種でも別物になる
- **配合発見**: 既知モンスター同士から意外な系譜が出ることで「数以上の広がり」を作る
- **エンドゲーム差分**: 裏トーナメントや深層ダンジョンでしか価値が見えない種を置く

### MVP の切り出し方
- MVP では 30体に絞る
- 30体は 5世界の生態系と配合導線を成立させる最小セットとして選ぶ
- 各体は将来の 400体体系の原型として設計し、後から拡張しても破綻しない family / rank / skill の基準点にする
- 「30体しかいない」ではなく、「30体でも配合したくなる」ことを優先する

### デザイン原則

#### 1. モチーフの明確さ
- 各モンスターは現実の生物、神話、民俗学、自然現象、人工物等から明確なモチーフを持つ
- モチーフが被る場合は解釈の角度を変える（例: 同じ「狼」モチーフでも、一方は北欧神話のフェンリル的解釈、他方は日本の大神的解釈）
- モチーフの出典を記録し、パクリにならないようガードレールを設ける

#### 2. シルエットの独自性
- ドット絵の小さなキャンバスでも、シルエットだけで判別できること
- 同族内でもシルエットの差別化を意識する
- 「丸い」「角ばった」「細長い」「左右非対称」等、シルエット言語を定義

#### 3. 世界観との整合性
- そのモンスターが「なぜその世界に生息しているのか」の理由がある
- 生態系における役割（捕食者/被食者/共生者/分解者 等）
- 人間社会との関わり（家畜化/脅威/信仰対象/素材源 等）

#### 4. ランクの視覚的説得力
- Eランクは小さくシンプル、Sランクは大きく複雑
- 同じ種族でもランクが上がると「より洗練された」デザインに
- Sランクは一目見て「こいつはヤバい」と感じるオーラ

#### 5. 配合の系譜が見えるデザイン
- 親の特徴が子に引き継がれるデザインルール
- 「ああ、この角は○○の血が入ってるな」とわかる視覚的手がかり
- 変異種は通常種と明確に区別できるが、同一モンスターだと認識できる

## 4.3 デザインルール（ピクセルアート仕様）

### スプライトサイズ
| ランク | バトルスプライト | フィールドスプライト |
|--------|----------------|-------------------|
| E | 24×24px | 16×16px |
| D | 32×32px | 16×16px |
| C | 32×32px | 16×16px |
| B | 48×48px | 16×16px |
| A | 48×48px | 16×16px |
| S | 56×56px | 16×16px（特殊演出あり） |

### カラールール
- **パレット制限**: GBC inspired 32色マスターパレットから選択
- **1スプライトあたりの色数**: ランクに応じた段階制（style bible 準拠）
  - 序盤（E〜D）: 4〜6色
  - 中盤（C〜B）: 5〜8色
  - 上位（A〜S）: 6〜10色
  - ※ GBCオリジナルでは1スプライト3色+透過（最大4色）だったが、本プロジェクトはGBC inspired であり、品質とランク表現のため緩和している
- **アウトライン**: 1pxアウトライン必須。原則は pure black `#000000` だが、style bible / production manual で許可された場合は **マスターパレット上の最暗色** を使ってよい
- **シェーディング**: 2トーンセルシェーディング（ベース色+影色）
- **禁止事項**:
  - アンチエイリアシング不可
  - ディザリング不可
  - グラデーション不可
  - サブピクセル表現不可

### アニメーション仕様
| 用途 | フレーム数 | フレーム間隔 |
|------|-----------|-------------|
| 待機（バトル） | 2フレーム | 500ms |
| 攻撃（バトル） | 3フレーム（Sランクのみ） | 150ms |
| 被ダメージ | 1フレーム（フラッシュ演出） | — |
| フィールド歩行 | 2フレーム × 4方向 | 200ms |

### デザインの禁忌（ガードレール）
1. **既存IPに似せない**: DQM、ポケモン、デジモン等のモンスターに酷似するデザインは禁止
2. **商標/著作権チェック**: デザイン確定前に類似キャラクターの調査を行う
3. **不快感の閾値**: ダークファンタジーだがグロテスク過ぎない。GBC時代の表現範囲を基準に
4. **文化的配慮**: 特定の宗教・民族を侮辱するデザインを避ける
5. **一貫性**: 同じ世界の同じ種族は同じ美術様式で描く

## 4.6 モンスター生態系設計

### 生態系の原則
> 「400体のモンスターは、20以上の世界にまたがる一つの巨大な生態系を形成する。」

### 生態系要素

#### 食物連鎖
- 各世界で食物連鎖を定義
- 下位ランクが上位ランクの餌になる自然な関係
- 食物連鎖の頂点に立つモンスターがその世界のボス的存在

#### 共生関係
- モンスター同士の共生（例: プラント系がスライム系に養分を提供し、スライム系がプラント系を外敵から守る）
- 人間との共生（家畜化されたモンスター、番犬的モンスター）
- 共生関係がゲームプレイに影響（特定の組み合わせでパーティボーナス等、要検討）

#### 縄張り・生息域
- 各モンスターに固有の生息世界・ダンジョン・階層を設定
- 希少モンスターは限られた場所にのみ出現
- 時間帯・天候による出現変動

#### 繁殖と配合の関係
- 自然界での繁殖（野生モンスターの世代交代）
- 人間による配合（人為的な種の操作）
- この二つの違いが世界観に組み込まれる（配合は「自然な繁殖」ではない）

### モンスター間の社会構造
| 構造 | 説明 | 例 |
|------|------|-----|
| **群れ型** | リーダーを中心に群れで行動 | ビースト系の一部 |
| **単独型** | 縄張りを持ち単独行動 | ドラゴン系上位 |
| **寄生型** | 他のモンスターに依存 | アンデッド系の一部 |
| **共生型** | 異種間で互いに利益 | プラント×スライム |
| **階層型** | 厳格なヒエラルキー | ディバイン系 |
| **遊牧型** | 世界を渡り歩く | 一部の希少種 |

## 4.7 変異種設計

### 変異種の種類

#### 色違い変異
- 通常種と同じデザインだがカラーパレットが異なる
- ステータス傾向が微変化
- 全400体に色違い変異の可能性あり

#### 形態変異
- デザインの一部が変化（角が増える、翼の形が変わる等）
- 固有スキルツリーを持つ
- 特定の配合条件でのみ発生

#### 覚醒変異
- 極めて稀（発生率1%以下）
- 通常種の上位互換的存在
- 専用の図鑑枠は持たない（通常種の図鑑内に記録）
- エンドコンテンツ向けの最強個体

### 変異の記録
- 変異種はセーブデータに個体情報として保存
- 図鑑には「変異種発見」のフラグが立つ
- 変異種のコレクションもやり込み要素の一つ

## 4.8 モンスターデータ定義テンプレート

各モンスターは以下のデータを持つ:

```yaml
monster:
  # === 基本情報 ===
  id: "M-001"
  name_jp: ""
  name_en: ""
  family: ""           # 種族（9種族のいずれか）
  element: ""          # 属性（8属性のいずれか）
  rank: ""             # ランク（E〜S）
  growth_type: ""      # 成長タイプ（早熟/普通/晩成/特殊）
  sprite_size: ""      # バトルスプライトサイズ

  # === ステータス（Lv1基準） ===
  stats:
    hp: 0
    mp: 0
    atk: 0
    def: 0
    spd: 0
    int: 0
    res: 0

  # === 属性耐性 ===
  resistances:
    fire: 1.0      # 0=吸収, 0.5=耐性, 1.0=通常, 1.5=弱点, 2.0=大弱点
    water: 1.0
    wind: 1.0
    earth: 1.0
    thunder: 1.0
    light: 1.0
    dark: 1.0

  # === 状態異常耐性 ===
  status_resistances:
    poison: 1.0     # 0=無効, 0.5=半減, 1.0=通常
    paralysis: 1.0
    sleep: 1.0
    confusion: 1.0
    curse: 1.0
    seal: 1.0
    instant_death: 1.0
    fear: 1.0

  # === スキルツリー ===
  skill_trees:
    innate: ""        # 種族固有ツリー
    slot_2: ""        # 第2ツリー（初期）
    slot_3: ""        # 第3ツリー（初期）
    # 配合で継承可能なツリーは制限なし（選択は3つまで）

  # === デザイン情報 ===
  design:
    motif: ""                    # モチーフ（何をベースにしたか）
    motif_source: ""             # モチーフの出典（神話名、文化圏等）
    silhouette_type: ""          # シルエット分類
    color_scheme: []             # 使用色（パレット番号）
    design_notes: ""             # デザイン上の注意点
    ip_check: ""                 # 類似キャラチェック結果

  # === 世界観情報 ===
  lore:
    habitat: []                  # 生息世界・ダンジョン
    ecology_role: ""             # 生態系における役割
    social_structure: ""         # 社会構造
    relationship_with_humans: "" # 人間との関係
    relationship_with_monsters: [] # 他モンスターとの関係
    history: ""                  # 歴史的背景
    dark_secret: ""              # 裏設定
    description_jp: ""           # 図鑑テキスト（日本語）
    description_en: ""           # 図鑑テキスト（英語）

  # === 配合情報 ===
  breeding:
    obtainable_by_scout: true/false   # スカウト入手可能か
    scout_locations: []               # スカウト可能な場所
    breeding_recipes: []              # この種を生み出す配合レシピ
    special_recipes: []               # 特殊配合レシピ
    mutation_from: ""                 # 変異元（変異種の場合）
    mutation_probability: 0.0         # 変異確率

  # === AI生成情報 ===
  generation:
    prompt_template: ""          # 生成用プロンプトテンプレートID
    prompt_overrides: ""         # モンスター固有のプロンプト補足
    generated_by: ""             # 使用ツール（niji7, NanoBanana等）
    version: ""                  # 生成バージョン
    approved: false              # 承認済みフラグ
```

## 4.10 デザインパイプライン（モンスター1体あたり）

### ワークフロー
```
1. コンセプト設計
   - モチーフ決定、世界観との紐付け、生態系での位置づけ
   - データテンプレート記入（lore, design セクション）

2. デザインプロンプト作成
   - style-bibleのルール + モンスター固有情報からプロンプト生成
   - パレット指定、サイズ指定、禁止事項を含む

3. AI生成（キャラデザ）
   - niji 7 でコンセプトアート生成
   - 複数候補から選定

4. スプライトシート生成
   - Nano Banana で3×3グリッド（9ポーズ）生成
   - バトル用・フィールド用それぞれ

5. アニメーション生成
   - Grok でスプライトシートからアニメーション生成

6. レビュー・修正
   - style-bible準拠チェック
   - パレット正規化（palette_remap.py）
   - IP類似チェック
   - 手動修正（必要な場合）

7. Godot統合
   - AtlasTexture化
   - SpriteFrames / AnimationPlayer設定
   - Resource（.tres）紐付け

8. 承認
   - 最終レビュー → approved フラグを true に
```

### 品質チェックリスト（1体ごと）
- [ ] モチーフが明確で、出典が記録されている
- [ ] 既存IPに酷似していない
- [ ] シルエットが同族内で差別化されている
- [ ] パレットがマスターパレット内に収まっている
- [ ] アウトラインが 1px で統一され、色は pure black または許可された最暗色ルールに従っている
- [ ] ランクに見合ったサイズ・複雑さ
- [ ] アニメーションがスムーズに再生される
- [ ] 世界観・生態系の記述が完成している
- [ ] 配合テーブルに正しく組み込まれている
- [ ] 図鑑テキスト（日英）が完成している
