## 2.1 コアループ概要

```
探索（ダンジョン/世界）
    ↓
遭遇・バトル（3v3ターン制）
    ↓
捕獲・スカウト
    ↓
育成（レベル上げ・スキル選択）
    ↓
配合（新モンスター誕生・変異種）
    ↓
パーティ再編成
    ↓
より強い敵・新世界への挑戦
    ↓
トーナメント参加
    ↓
ストーリー進行・世界の真実に迫る
    ↓
（繰り返し）
```

### プレイセッション設計
- **短セッション（5〜15分）**: ダンジョン1フロア探索、バトル数回、配合1回
- **中セッション（30〜60分）**: ダンジョン攻略、トーナメント1ランク、ストーリーイベント
- **長セッション（1時間〜）**: 新世界攻略、配合ルート研究、裏コンテンツ挑戦

### 配合中毒を生む設計原則
- コアループは **予期 → 行動 → 報酬 → 喪失 → 再構築** の循環で設計する
- 配合は「親が消える」痛みを持たせつつ、継承値、スキル、履歴で投資が無駄にならない形にする
- 3体パーティの制限で「何を連れていくか」の選択圧を高め、モンスターを単なる所持品ではなく相棒にする
- ハブ、メニュー、配合画面は短い導線で回せるようにし、「次の一回」にすぐ入れるテンポを守る
- 特殊配合、断片ヒント、図鑑文で「自分で発見した」感覚を作り、ノートを取りたくなる深さを目指す
- **不可逆性と救済**、**準備の重み**、**局所ランダム / 長期決定論** を同時に成立させる

## 2.2 バトルシステム

### 基本仕様
| 項目 | 仕様 |
|------|------|
| **形式** | ターン制コマンドバトル |
| **パーティ構成** | 3v3（バトル参加3体。控えはなし、牧場で管理） |
| **行動順** | 素早さ基準 + ランダム幅（-5〜+5） |
| **コマンド文法** | たたかう / さくせん / どうぐ / にげる |

### コマンド文法
- 標準体験は **短い4コマンド** を維持し、1ターンごとの細かい手入力よりも、事前に組んだ作戦と育成差が勝敗を分ける形にする
- `たたかう` は各モンスターの現在作戦に従って行動を実行する
- `さくせん` で個体別の方針を変更できる
- `どうぐ` では回復、補助、勧誘用アイテムを扱う
- `めいれいさせろ` は作戦の一種として許可し、要所ではプレイヤーが直接入力できる
- これにより「普段は高速、難所では介入可能」というリズムを作る

### 作戦システム（AI制御）
パーティメンバーに個別の作戦を設定可能（DQM準拠）:

| 作戦 | 挙動 |
|------|------|
| **ガンガンいこうぜ** | 攻撃特化。MP消費を惜しまず最大火力 |
| **いのちだいじに** | 回復優先。HPが減ったら即回復 |
| **じゅもんつかうな** | 通常攻撃のみ。MP温存 |
| **めいれいさせろ** | プレイヤーが直接コマンド選択 |
| **バッチリがんばれ** | バランス型。状況に応じてAI判断 |
| **サポートたのむぞ** | 補助魔法・デバフ優先 |

### ダメージ計算

#### 物理ダメージ
```
基礎ダメージ = (攻撃力 × 0.5) - (防御力 × 0.25)
最終ダメージ = 基礎ダメージ × ランダム幅(0.85〜1.0) × 属性倍率 × 会心倍率
```

#### 呪文ダメージ
```
基礎ダメージ = 技ごとの固定威力帯
最終ダメージ = 基礎ダメージ × ランダム幅(0.9〜1.1) × 属性倍率 × 耐性補正
```

### 賢さ・精神の役割
- **賢さ** は攻撃呪文の素ダメージを直接伸ばしすぎない
- 賢さは主に **補助呪文の成功率 / 状態異常の通りやすさ / AIの判断精度** に影響する
- **精神** は属性耐性、状態異常耐性、MP関連補正に寄せる
- これにより「呪文火力を上げるために賢さだけを盛る」単調な最適解を避け、準備と相性読みを重くする

#### 特殊倍率
| 要素 | 倍率 |
|------|------|
| **会心の一撃** | 1.5倍（発生率: 1/16 = 6.25%） |
| **ぼうぎょ時** | 被ダメージ 0.5倍 |
| **属性弱点** | 1.5倍（要調整） |
| **属性耐性** | 0.5倍（要調整） |
| **属性無効** | 0倍 |
| **属性吸収** | -0.5倍（回復に転換、要検討） |

### 属性システム
| 属性 | 対応色イメージ |
|------|---------------|
| なし | 灰 |
| 火 | 赤 |
| 水 | 青 |
| 風 | 緑 |
| 地 | 茶 |
| 雷 | 黄 |
| 光 | 白 |
| 闇 | 紫 |

属性相性表は別途詳細設計（04_monster_design.md にて）。

### 状態異常
| 状態 | 効果 | 持続 |
|------|------|------|
| 毒 | 毎ターンHP減少（最大HPの1/16） | 戦闘終了まで or 回復 |
| 麻痺 | 1/4の確率で行動不能 | 数ターン |
| 眠り | 行動不能。被ダメージで解除 | 数ターン |
| 混乱 | ランダム対象に攻撃（味方含む） | 数ターン |
| 呪い | 攻撃時に自分もダメージ | 戦闘終了まで or 回復 |
| 封印 | 特技使用不可 | 数ターン |
| 即死 | HP0。一定確率で耐性貫通 | — |
| 恐怖 | 行動キャンセル率上昇 | 数ターン |

### 逃走判定
```
逃走成功率 = (パーティ平均素早さ / 敵平均素早さ) × 50%
最低保証: 10%
ボス戦: 逃走不可
```

### 経験値配分
- 戦闘参加モンスター全員に均等配分
- 控えには配分なし（参加インセンティブ）
- ボスモンスターは通常の3〜5倍の経験値

## 2.3 モンスター捕獲・スカウト

### スカウトシステム（DQM寄りの勧誘方式）
- `スカウト` は専用コマンドの連打ではなく、**勧誘用アイテム / 特技 / 戦闘中の振る舞い全体** を含む加入システムとして扱う
- 基本導線は `どうぐ` から勧誘用アイテムを使う、または一部モンスター固有の勧誘特技を使う形にする
- 倒した敵のうち **最後に印象を残した対象** が加入候補になる
- 成功率は複数要素で決定:

```
基礎加入率 = 敵の残HP割合補正 × 状態補正 × アイテム補正 × レベル差補正
状態補正: 通常=1.0, 毒=1.1, 麻痺=1.2, 眠り=1.35
アイテム補正: 勧誘用アイテムの種類で変動
レベル差補正: 自パーティ平均Lv > 敵Lv なら上昇、逆なら下降
```

- **ランク補正**: 高ランクモンスターほどスカウト困難
  - Eランク: 基礎率 ×1.5
  - Dランク: 基礎率 ×1.2
  - Cランク: 基礎率 ×1.0
  - Bランク: 基礎率 ×0.6
  - Aランク: 基礎率 ×0.3
  - Sランク: スカウト不可（配合でのみ入手）

- **同種モンスター所持時ペナルティ**: 成功率半減
- **1バトル中の試行回数制限**: 3回まで
- **失敗時**: 敵が怒り状態（攻撃力1.2倍）
- **表示方針**: 成功率を数字で露骨に見せず、「かなり気を許した」「まだ警戒している」など文言で手応えを返す
- **設計原則**: 確率の揺らぎはここに閉じ込め、主要進行と配合の目標達成は決定論で支える

## 2.4 育成システム

### レベルシステム
| 項目 | 仕様 |
|------|------|
| **最大レベル** | 99 |
| **経験値曲線** | 成長タイプにより4種（早熟/普通/晩成/特殊） |
| **ステータス** | HP, MP, 攻撃力, 防御力, 素早さ, 賢さ, 精神 |

### 成長タイプ
| タイプ | 特徴 | 代表モンスター例 |
|--------|------|-----------------|
| **早熟** | 序盤は強いが後半伸び悩む | Eランク雑魚系 |
| **普通** | 均等に成長 | 中堅モンスター |
| **晩成** | 序盤弱いが後半爆発的に成長 | Aランク系 |
| **特殊** | 特定ステータスだけ極端に伸びる | Sランク・ボス系 |

### スキルツリーシステム
- 各モンスターは最大**3つのスキルツリー**を持てる
- スキルツリーはレベルアップ時にスキルポイント（SP）を割り振って解放
- 同じモンスターでもSPの振り方で全く違う技構成になる
- **技の上限**: 1体が同時に使える技は最大**8つ**
- 8つを超えた場合、どの技を残すか選択（DQM方式）

### スキルツリーの種類（カテゴリ例）
| カテゴリ | 内容 |
|----------|------|
| **種族固有** | そのモンスター固有の技系統 |
| **属性系** | 火の息、氷の息、雷撃 等 |
| **回復系** | ホイミ系、リジェネ系 |
| **補助系** | バフ、デバフ、状態異常付与 |
| **物理系** | 連続攻撃、範囲攻撃 |
| **特殊系** | 配合でのみ継承される希少ツリー |

### 性格システム
| 性格 | 効果 |
|------|------|
| **攻撃的** | ATK +10%, DEF -5% |
| **慎重** | DEF +10%, SPD -5% |
| **臆病** | SPD +10%, ATK -5% |
| **野生的** | 会心率 +3%, 命令無視率 +5% |
| **知的** | INT +10%, HP -5% |
| **穏やか** | RES +10%, ATK -5% |
| **気まぐれ** | 全ステ ±5%（毎バトルランダム） |
| **頑固** | DEF +15%, 命令無視率 +10% |

### 忠誠度・なつき度
- 0〜255の範囲
- 影響: 命令遵守率、会心率ボーナス、配合時の特殊レシピ解放
- 上昇条件: バトル勝利、アイテム使用、一緒に冒険した時間
- 下降条件: 戦闘不能放置、パーティから長期外す

## 2.5 配合システム

### 採用モデル
- ベースは **家系法則 + 特殊レシピ + 変異種** のハイブリッド
- 総当たりの完全個別表にはしない
- 覚えきれない深さは、特殊レシピ、隠し条件、多段配合、変異条件で担保する

### 基本ルール
- 親2体を掛け合わせて新モンスターを生む
- **両親はLv10以上**が必要
- 配合後、親は消滅（重要な決断）
- 生まれた子は**Lv1**からスタート
- 子のステータスには親のステータスの一部が継承される
- 親が消えても「世代が前進した」と感じられるよう、継承結果は必ず可視化する

### 配合で守る感情設計
- 親消滅はショックであってよいが、徒労感にはしない
- 子には **配合履歴 / 継承表示 / 変異履歴** を持たせ、「この個体は自分が作った」と感じさせる
- 同じ種でも技、性格、継承値、履歴で別個体として成立するようにする
- 強モンスターの最短レシピだけを追うのでなく、「今いる個体をどう次世代へ託すか」が主目的になるよう調整する

### ステータス継承
```
子の基礎ステータス += (親1のステータス + 親2のステータス) / 2 × 1/8
```

### 配合回数ボーナス
- 配合を重ねるほど子の基礎ステータスにボーナス
- **1回の配合につき +2%**（最大 +50% = 25世代）
- つまり、何世代にもわたって配合を重ねた血統ほど強くなる

### スキルツリー継承
- 親が持つスキルツリーから最大3つを選んで子に継承
- 配合でしか手に入らない**特殊スキルツリー**が存在
- 特定の組み合わせでのみ解放される**隠しスキルツリー**

### 配合テーブル構造

#### 家系配合（Family Breeding）
9つの種族 × 9つの種族 = 81パターンの基本テーブル

| 親1＼親2 | スライム | ビースト | バード | プラント | マジック | マテリアル | アンデッド | ドラゴン | ディバイン |
|----------|---------|---------|--------|---------|---------|-----------|-----------|---------|-----------|
| スライム | スライム | ビースト | バード | スライム | マジック | マテリアル | スライム | ドラゴン | ディバイン |
| ビースト | ビースト | ビースト | バード | プラント | マジック | ビースト | アンデッド | ドラゴン | ディバイン |
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

（完全テーブルは別途マスターデータとして定義）

#### 特殊配合（Special Breeding）
- 特定のモンスターID × 特定のモンスターID = 特定の結果
- 家系配合より優先される
- 約200〜400パターン（400体に対応する複雑さ）
- 一部は隠しレシピ（ゲーム内にヒントを散りばめる）

#### 変異種（Mutation）
- 配合時に**一定確率**で通常結果と異なるモンスターが誕生
- 変異確率: 基本 **3%**
- 特定条件で変異確率上昇:
  - 特定アイテム使用: +5%
  - 特定の場所で配合: +3%
  - 親の配合回数が多い: +1%/世代（最大+10%）
  - 特定の月齢（ゲーム内時間）: +2%
- 変異種は通常種と同じ基本ステータスだが、以下が異なる:
  - **固有カラーリング**（色違い）
  - **固有スキルツリー**（変異種限定）
  - **ステータス傾向の変化**（例: ATK型→INT型に変異）
- 変異種からさらに配合すると、変異特性が遺伝する可能性あり

### 配合の複雑さを担保する設計
1. **多段配合**: A+B=C、C+D=E のように、目的のモンスターに到達するまで複数回の配合が必要
2. **循環依存**: 一部のレシピは「このモンスターを作るにはあのモンスターが必要、あのモンスターを作るにはこのモンスターが必要」という構造
3. **素材の希少性**: 特定モンスターは特定ダンジョンの特定階層にしか出現しない
4. **配合条件**: 一部の特殊配合はレベル、忠誠度、特定スキル習得等の追加条件あり
5. **世代管理**: 「3世代以上配合を重ねた○○族」のような条件

## 2.6 トーナメントシステム

### ランク構成
| ランク | レベル上限 | 対戦数 | 報酬 |
|--------|----------|--------|------|
| Gランク | Lv10 | 4 | 序盤素材、金 |
| Fランク | Lv15 | 4 | 配合アイテム |
| Eランク | Lv20 | 4 | スキルツリー解放 |
| Dランク | Lv30 | 4 | 希少モンスター情報 |
| Cランク | Lv40 | 5 | 新世界への鍵 |
| Bランク | Lv50 | 5 | 特殊配合レシピ |
| Aランク | Lv70 | 6 | 裏ダンジョン解放 |
| スターフォール | Lv99 | 8 | エンディング条件の一つ |

### トーナメント設計方針
- レベル上限制により、レベルゴリ押しではなく**配合と戦略**が問われる
- 対戦相手はランクに応じたAIで生成
- 一部の対戦相手は固定キャラ（ストーリーと連動）
- 敗北してもペナルティなし（再挑戦可能）
- ランクが上がると新世界やストーリーが解放される連動設計

- 並び替え（種族、レベル、入手順等）

## 2.8 アイテムシステム

### カテゴリ
| カテゴリ | 用途 |
|----------|------|
| **回復系** | HP/MP回復、状態異常治療 |
| **バトル系** | 一時的バフ、敵へのデバフ |
| **配合系** | 変異確率上昇、特殊配合の触媒 |
| **探索系** | ダンジョン脱出、マップ表示 |
| **キーアイテム** | ストーリー進行、新世界解放 |
| **素材系** | NPCへの納品、特殊アイテム作成 |

### 所持制限
- 携行枠上限: **20枠**
- 同一アイテム所持数上限: 99個
- キーアイテムは別枠管理

### 所持制限の設計原則
- 所持制限は不便のためではなく、**何を持つかをゲームにする**ために置く
- Vertical Slice では **携行20枠前後** を基準に検証し、テンポと判断量を計測する
- 倉庫/牧場を高速導線にして、「持てない不満」ではなく「選ぶ面白さ」に寄せる

## 2.9 ゲーム内時間・天候

### 時間システム
- ゲーム内に昼夜サイクルあり（フィールド歩数 or リアル時間連動、要検討）
- 時間帯によって出現モンスターが変わる
- 特定イベントは特定時間帯のみ発生

### 天候システム（要検討）
- 世界ごとに天候パターン
- 天候が属性ボーナスに影響（雨天時は水属性+10%等）
- 特定天候でのみ出現するモンスター

## 2.10 難易度設計方針

### 基本方針: ガチ寄り
- 配合・スキル構成を考えないとボスに勝てない
- レベルだけでゴリ押しできないバランス（レベル上限、属性相性の重要度）
- 探索では回復リソース管理が鍵
- 全滅時のペナルティ: 所持金半減 + 拠点送還（DQ方式）

### 難易度曲線
```
序盤（世界1〜5）: チュートリアル兼ねた導入。基礎を学ぶ
中盤（世界6〜12）: 配合の重要性を体感。属性相性が死活問題に
終盤（世界13〜20）: 多段配合・変異種なしではクリア困難
裏（エンドコンテンツ）: ガチ勢でも唸る高難度
```
