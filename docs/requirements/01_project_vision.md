## 1.2 プロダクトビジョン

### ワンライナー
> 「45歳のおっさん/おばさんが主人公の、生きた門をくぐって20以上の世界を渡り歩くダークファンタジー・モンスター育成RPG。GBCの手触りと現代UXの利便性を融合し、1000時間溶かせる配合の深淵を提供する。」

### ビジョンステートメント

本作は、DQM・ポケモン・メダロット・風来のシレンといったレトロゲームの黄金期に感じた「ワクワク」を、現代のAI技術を駆使して一人で再現するプロジェクトである。

安易なレトロ風ゲームではなく、任天堂・スクエニがプロとして製品を出荷するのと同じ水準の設計・品質管理プロセスを経て、丁寧に作り込む。

## 1.3 コアバリュー（絶対に妥協しないもの）

### 1. 配合の深淵
- 簡単に覚えられないレベルの複雑な配合体系
- 変異種のランダム発生による予測不能な面白さ
- 「このルート見つけた！」という発見の喜び

### 2. モンスターへの愛着
- 同じモンスターでもスキル構成・育成方針で唯一の存在になる
- 個体ごとの特別感（性格、忠誠度、配合履歴）
- 「こいつを手放したくない」と思わせる設計

### 3. 世界の深み
- 表面は美しいファンタジー、裏にはドロドロした闇
- 1000年単位の歴史が全てのモンスター・場所・NPCに紐づく
- 世界間移動そのものが世界の禁忌とモンスター存在論に接続している
- 「終わったと思ったらまだある」多層構造のストーリー

### 4. 丁寧な作り込み
- 安っぽいデザイン・安易な設計を絶対にしない
- GBCのピクセルアート美学を正しく理解し、忠実に再現
- プロの開発フローに則った品質保証

### 体験設計哲学
| 抽象原則 | 具体ルール |
|----------|-----------|
| **自分だけの個体を作っている感覚を最優先する** | 配合、継承、履歴、個体差で「自分が作った」手触りを残す |
| **不可逆性と救済を同居させる** | 親消滅や所持制限で決断の重みを出しつつ、継承・牧場・再挑戦で徒労感を防ぐ |
| **勝敗は準備で決まる** | 戦闘は短く、配合、作戦、所持品整理、属性対策の比重を高くする |
| **確率は局所、長期目標は決定論** | 捕獲、変異、ドロップには揺らぎを残し、狙い種や主要進行は計画可能にする |
| **デフォルトは当時感、オプションで現代快適** | 160×144 / 20×18 / 8×8 の情報密度を守り、速度や詳細表示は設定で拡張する |
| **既存IPの表現はなぞらず、原理だけ継ぐ** | ループや制約は学ぶが、固有名詞、造形、文言、UI意匠は独自化する |

## 1.4 ターゲットユーザー

### プライマリ
- **レトロゲーマー**（25〜40歳）
- DQM、ポケモン金銀、メダロット等をリアルタイムでプレイした世代
- 「あの頃のワクワク」を求めている
- やり込み要素に時間を惜しまない

### セカンダリ
- ドット絵・ピクセルアート愛好家
- モンスター育成ジャンルのファン全般
- ダークファンタジー好き

### ユーザーペルソナ
> **田中（35歳・会社員）**: 小学生の時にDQM2を死ぬほどやり込んだ。最近のソシャゲには飽きている。通勤電車で腰を据えて遊べる、「ちゃんとしたRPG」を探している。配合で最強パーティを作るのが生きがい。

## 1.5 スコープ定義

### スコープは4層で管理する
- **North Star**: 最終的に実現したい完全版ビジョン。現時点の大目標。
- **Vertical Slice**: コアループが成立しているかを判定する試作範囲。
- **MVP**: Vertical Slice を超えて、作品の方向性と量産フローを検証する中間目標。
- **Initial Release**: 初回出荷の実スコープ。Phase 0〜1の検証結果を見て確定する。本番品質を優先し、North Star にできるだけ近づける。

### North Star（完全版ビジョン）
- メインストーリー全編（60〜80時間）
- 20以上の世界（各世界に固有の文化・生態系・歴史）
- モンスター400体（配合テーブル・変異種含む）
- ターン制バトル（3v3）
- 配合システム（家系テーブル + 特殊レシピ + 変異種のハイブリッド）
- スキルカスタマイズ（技の取捨選択・継承）
- トーナメント（8ランク）
- ランダム生成ダンジョン
- 裏ボス・裏ダンジョン・エンドコンテンツ
- 日本語 / 英語対応
- iCloudセーブ同期
- バーチャルパッド操作

### Vertical Slice（Phase 1の必達）
- 1つの世界 or 拠点+周辺1マップ
- モンスター10体
- フィールド移動、エンカウント、3v3バトル
- スカウト、育成、基本配合
- プレースホルダーアセットで5分間遊べる状態

### MVP（中間マイルストーン）
- 世界数: 5
- モンスター数: 30
- 言語対応: 日本語先行
- 目的: 世界観、配合導線、アセット量産、テキスト運用、実装速度の現実値を確認する
- 注意: これは初回出荷スコープではなく、本番前の中間目標

### Initial Release を確定するゲート
- Godot iOSエクスポートからTestFlight配布まで一通り通る
- ローカルセーブ、オートセーブ、異常終了復帰が安定する
- iCloud / GDExtension の技術スパイク結果が出る
- 1世界あたりのテキスト量、アセット量、実装工数の予実が見える
- 上記が揃うまで、言語対応範囲は「目標」であり「初回出荷の確約」ではない

### Initial Release の原則
- North Star の価値をなるべく損なわない本番品質を優先する
- 追加しやすいデータ駆動設計を先に固め、段階的拡張に耐える構造を維持する
- 反応の良いコアループ、愛着の湧く育成、印象に残るダークファンタジー表現を最優先にする
- 初回出荷後に世界、モンスター、言語、エンドコンテンツを段階的に拡張できる構造を維持する
- 初回リリース時点でメインストーリーは一本の作品として完結させ、続編前提の切り方はしない

### Initial Release で確定している事項
- 世界数: 20以上
- モンスター数: 400体
- 言語対応: 日本語先行
- メインストーリー: 初回リリース時点で完結必須
- エンドコンテンツ: 裏ボス、裏ダンジョン、無限ダンジョンを含める
- iCloud同期: 実装コストが重ければ初回リリースから外してよい
- 方針: MVP とは別に、本番版として成立するボリュームと品質を持たせる

### Initial Release から外す前提のもの
- オンライン対戦・交換
- 追加世界（DLC的拡張）
- 他プラットフォーム対応（Android, Steam等）

## 1.7 タイトル決定方針

正式タイトルは未定。以下の方向性で検討する：

### タイトルに求める要素
- モンスター育成を想起させる
- ダークファンタジーの世界観を匂わせる
- 覚えやすく、検索しやすい
- 日英両方で違和感がない
- 既存IPと被らない（商標チェック必須）

### 決定プロセス
1. 世界観・ストーリーの骨格が固まった段階でタイトル案を複数出す
2. 商標調査
3. ドメイン・App Store名の空き確認
4. 最終決定
