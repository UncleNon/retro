## 5.2 世界構成設計

### MVP の考え方
- MVP では 5世界に絞る
- 5世界はそれぞれ明確に異なる文化・生態系・禁忌を持たせる
- 将来の 20+世界構想の縮小版ではなく、「5世界でも一本の作品として成立する構成」を優先する
- 追加世界は後続アップデートで自然に接続できるよう、中央ハブと生きた門のID体系は最初から拡張可能にしておく

### Initial Release の考え方
- 初回リリースでは 20以上の世界を入れる
- MVP で固めた世界設計テンプレートと量産フローを、20+世界へ拡張する
- 数だけ揃えるのではなく、各世界に文化・生態系・禁忌・導線上の役割を持たせる
- 世界間移動は全て中央ハブを基点に設計し、世界解放そのものを進行のご褒美にする

### 世界のカテゴリ分類

各世界は自然環境テーマを持ち、そのテーマがモンスター生態系・文化・ダンジョン設計に直結する。

| カテゴリ | 世界数目安 | テーマ例 |
|----------|----------|---------|
| **自然型** | 6〜8 | 森林、草原、砂漠、雪原、火山、海洋、湿地、山岳 |
| **文明型** | 5〜7 | 王国、帝国、遺跡都市、地下都市、浮遊都市、商業都市 |
| **異質型** | 4〜5 | 夢の世界、鏡の世界、時間停止世界、闇の世界、虚無 |
| **隠し型** | 3〜5 | エンドコンテンツ専用、特殊条件で解放 |

### 世界の解放順序

```
ストーリー序盤（世界1〜5）
├── 世界1: チュートリアル世界（小さな村、塔周辺、基礎学習）
├── 世界2: 最初の冒険先（配合を学ぶ）
├── 世界3: 属性相性を学ぶ
├── 世界4: トーナメント初参加
└── 世界5: 最初の「闇」を垣間見る

ストーリー中盤（世界6〜12）
├── 世界6〜8: 各世界の問題を解決しつつ真実に近づく
├── 世界9〜10: 世界観の根幹に触れる
└── 世界11〜12: 裏の存在が表面化

ストーリー終盤（世界13〜20）
├── 世界13〜16: 大きな対立構造の中で選択を迫られる
├── 世界17〜19: クライマックスへ向かう
└── 世界20: 最終決戦

エンドコンテンツ（隠し世界）
├── 裏世界A: 裏ストーリー
├── 裏世界B: 無限ダンジョン
└── 裏世界C〜: 最高難度コンテンツ
```

## 5.4 ランダム生成ダンジョン

### 設計方針
- BSP（Binary Space Partition）アルゴリズムベース（既存実装を発展）
- 毎回異なるフロア構成でリプレイ性を担保
- エンドコンテンツの核（無限ダンジョン）

### ランダムダンジョンの仕様
| 項目 | 仕様 |
|------|------|
| **フロア数** | 無限（深くなるほど難易度上昇） |
| **マップサイズ** | フロアごとに可変（深層ほど広い） |
| **モンスター出現** | 深さに応じたレベルテーブル |
| **アイテム** | ランダムドロップ、深層限定レア |
| **ボス出現** | 10フロアごとにミニボス |
| **脱出** | 専用アイテムで脱出可能。死亡時は所持アイテムロスト（要検討） |
| **シード値** | 記録可能（同じダンジョンを再挑戦可能にするか要検討） |

### 風来のシレン的要素（検討中）
- 持ち込み制限ダンジョン（Lv1から開始）
- アイテム未鑑定システム
- 罠システム

## 5.5 フィールド移動

### 基本仕様
| 項目 | 仕様 |
|------|------|
| **移動方式** | 4方向タイルベース移動 |
| **タイルサイズ** | 8×8px |
| **移動速度** | 1タイル/200ms（通常）、倍速対応 |
| **エンカウント** | ランダムエンカウント（歩数ベース） |
| **速度倍率** | 1倍/2倍/4倍（設定で切替） |

### 世界間移動
- 世界間移動は **中央ハブ + 生きた門** で固定する
- 中央ハブの仮称は **塔**。プレイヤーはここを起点に各世界へ渡る
- 塔はワールド外のロビーではなく、通常のフィールド上に存在する特異建築として配置する
- 各門は世界ごとに固有の見た目、開放条件、演出を持つ
- 解放条件はストーリー進行、トーナメントランク、キーアイテム、特定系統モンスターの共鳴などを組み合わせる
- 画面遷移は単純な一覧メニューではなく、「門に近づく → 反応条件を満たす → 越境演出 → 世界へ遷移」の流れを基本とする
- 門の情報は図鑑的に蓄積し、未解放門にも不穏な説明文や断片的ヒントを持たせる
- 塔の説明は最初から明かしすぎず、「聖域」「遺跡」「墓」「星見台」など複数の呼び名が併存する状態を許容する

### 塔の役割
| 要素 | 内容 |
|------|------|
| **機能** | 世界間移動の起点、情報集積、ストーリー進行の中枢 |
| **雰囲気** | 聖堂、修道院、天文台、墓所が混ざったように見えるが、何のための建物か断定できない冷たい塔 |
| **ゲーム上の役割** | 新世界解放の演出、門の状態変化、断片情報の蓄積、禁忌の匂わせ |
| **拡張性** | 20+世界に対応しやすく、追加門も自然に実装できる |
| **導線** | プレイヤーが次の目標と未解放の謎を同時に視認できる。通常世界の一部として行き来できる |

### 塔の導線原則
- 配合、回復、記録、情報取得、世界移動の主要導線は短く保つ
- 重要機能は初見でも迷わず辿れ、周回時にはほぼ儀式のように回せる配置にする
- 世界を越えて戻るたびに、塔の一部が微妙に変化しているように見せ、理解不能さを増幅させる

### マップギミック
| ギミック | 説明 |
|----------|------|
| **隠し通路** | 壁に見えるが通れる場所 |
| **スイッチ** | 踏むと道が開く |
| **ワープ** | 同フロア内の別地点に飛ぶ |
| **一方通行** | 戻れない通路（緊張感） |
| **環境ダメージ** | 溶岩、毒沼等（歩くとダメージ） |
| **滑る床** | 氷上で操作不能になる |
| **暗闇** | 視界が制限される |
| **イベントタイル** | 踏むとイベント発生 |

## 5.6 タイルセット設計

### タイルセット分類
| カテゴリ | 種類 |
|----------|------|
| **自然** | 森林、草原、砂漠、雪原、火山、海岸、湿地、山岳 |
| **洞窟** | 岩洞窟、氷洞窟、溶岩洞窟、水中洞窟、遺跡洞窟 |
| **建造物** | 城、町、村、神殿、塔、地下牢 |
| **異質** | 夢幻、鏡面、虚無、歪み |
| **共通** | 階段、扉、宝箱、看板、NPC用 |

### タイルセット仕様
- 8×8px基本グリッド
- 各タイルセットは最低64種類のタイル
- オートタイル対応（地形の境界を自動処理）
- レイヤー構成: 地面 / 壁・障害物 / 装飾 / イベント

### AI生成方針
- スタイルバイブルのルール（パレット、アウトライン等）に準拠したプロンプトで生成
- 世界のテーマに合わせたカラーバリエーション
- 手動で微調整（タイルの繋ぎ目、パターンの不自然さ修正）
