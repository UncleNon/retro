## 10.3 テキスト管理

### データ構造
```yaml
text_entry:
  id: "TXT-MAIN-001-001"     # カテゴリ-章-番号
  context: "第1章・最初の町のNPCとの会話"
  speaker: "NPC-001"          # 話者ID（該当する場合）
  ja: "日本語テキスト"
  en: ""                      # 英語対応後に投入
  notes: "翻訳時の注意点"
  status: "draft/reviewed/approved"
```

### テキストID体系
```
TXT-MAIN-{章番号}-{連番}     # メインストーリー
TXT-SUB-{世界ID}-{連番}      # サブクエスト
TXT-DARK-{章番号}-{連番}     # 裏ストーリー
TXT-NPC-{NPCID}-{連番}       # NPC会話
TXT-MON-{モンスターID}       # 図鑑テキスト
TXT-ITEM-{アイテムID}        # アイテム説明
TXT-SKILL-{スキルID}         # スキル説明
TXT-SYS-{カテゴリ}-{連番}    # システムメッセージ
```

## 10.4 AI生成フロー

### ストーリー・会話テキスト
```
1. シナリオ骨格（プロット）を人力で策定
2. シーンごとの目的・感情・伏線要素を定義
3. AI（Claude/GPT）でテキスト生成
   - キャラクターの口調・性格を指定
   - 世界観・トーンのガイドラインを提供
   - 伏線の配置指示
4. 人力レビュー
   - 世界観との整合性チェック
   - 伏線の配置確認
   - 不自然な表現の修正
5. MVP / Initial Release ともに日本語版を先に完成
6. 後続フェーズでローカライズ（日→英翻訳）
   - AI翻訳 + ニュアンス確認
```

### モンスター図鑑テキスト
```
1. モンスターデータシート（lore情報）をインプット
2. AIで図鑑テキスト生成
   - 生態系の情報を反映
   - 裏設定のヒントをさりげなく含める
   - 読み物としても面白いテキスト
3. MVP は日本語版のみ承認
4. 後続フェーズで日本語→英語翻訳
5. レビュー・承認
```

## 10.5 テキスト設計ガイドライン

### 文体
| 場面 | 文体 |
|------|------|
| **ナレーション** | 三人称、やや硬め、文語寄り |
| **NPC会話** | キャラクターに応じた口語 |
| **主人公** | 無言。選択肢、短い反応、演出で意思表示 |
| **図鑑** | 学術的だが読みやすい |
| **システム** | 簡潔、敬体 |

### テキスト品質基準
1. **安易なセリフを書かない**: 一文一文に意図を持たせる
2. **行間で語る**: 直接言わず、プレイヤーに推測させる
3. **世界に生活感**: NPCは「RPGのNPC」ではなく「その世界の住民」
4. **ダークさは示唆で**: グロテスクな描写より、想像を掻き立てる表現
5. **伏線は自然に**: 後から気づいて「あ！」となるレベル

### ローカライズ方針
- Initial Release は日本語先行で出荷し、英語対応は後続フェーズで追加する
- 直訳ではなく、英語として自然な表現
- 文化的なニュアンスが伝わるよう意訳を許容
- 固有名詞は日英で統一ルールを設定
- モンスター名は英語でも覚えやすく、世界観に合うもの

## 10.6 フォント要件

### 日本語フォント
- ひらがな、カタカナ、常用漢字（JIS第一水準目安）
- 8×8pxビットマップフォント
- GBC風の角張ったデザイン

### 英語フォント
- ASCII全文字
- 8×8pxビットマップフォント
- 日本語フォントとデザインの統一感

### 数字フォント
- 0〜9 + 記号（/, %, +, -, ×）
- ダメージ表示用の太字バリエーション

## 10.7 テキスト表示仕様

| 項目 | 仕様 |
|------|------|
| **表示速度** | 50ms/文字（デフォルト） |
| **即表示** | ボタン押しで残りテキスト即表示 |
| **改ページ** | ウィンドウ内に収まらない場合、▼カーソルで次ページ |
| **選択肢** | 最大4択、カーソル選択 |
| **名前表示** | ウィンドウ上部に話者名（該当時） |
