# 07. UI/UX設計

> **ステータス**: Draft v1.0
> **最終更新**: 2026-03-15

---

## 7.1 設計方針

> GBCのレトロ感をベースに、現代モバイルの利便性を融合する。見た目はレトロ、使い勝手はモダン。

### 原則
1. **GBC美学の忠実な再現**: ドット絵ウィンドウ、ビットマップフォント、ピクセルパーフェクト
2. **現代UXの利便性**: ワンタップアクセス、ソート/フィルター、クイックセーブ
3. **情報密度の最適化**: 160×144pxの画面に必要十分な情報を過不足なく
4. **操作の一貫性**: 全画面で同じ操作体系
5. **デフォルトは当時感、オプションで現代快適**: 便利機能は追加しても標準体験を壊さない

## 7.2 入力方式

### バーチャルパッド
| 要素 | 仕様 |
|------|------|
| **十字キー** | 画面左下固定、4方向（斜め入力なし） |
| **Aボタン** | 決定/話しかける/調べる |
| **Bボタン** | キャンセル/戻る |
| **メニューボタン** | メインメニュー開閉 |
| **スタートボタン** | ポーズ/システムメニュー |

### タッチ操作の補助（現代UX）
- メニュー内の項目はタップで直接選択可能
- モンスター一覧はスワイプでスクロール
- ピンチ操作は不使用（ピクセルパーフェクトを崩さない）

### 操作カスタマイズ
- ボタンサイズの調整（3段階）
- 透過度の調整
- 左手/右手の片手モードは v1 では非対応。将来検討

## 7.4 UI部品仕様

### 情報密度規格
- 基本グリッドは **20×18タイル / 1タイル8×8px**
- リスト系UIは、可能な限り画面下部 **2行** を説明帯として固定する
- 詳細情報は1画面に詰め込まず、必要時のみ別画面や詳細表示に逃がす
- テキスト速度はオプションで変更可能にし、標準は当時感を崩さない速度にする

### メッセージウィンドウ
| 項目 | 仕様 |
|------|------|
| **サイズ** | 画面下部、幅いっぱい × 高さ4行分 |
| **デザイン** | 9-sliceドット絵ウィンドウ（8×8pxコーナー） |
| **テキスト表示** | 1文字ずつ表示（50ms/文字）、タップで全文即表示 |
| **カーソル** | 8×8px、2フレームアニメーション（300ms） |
| **フォント** | ビットマップフォント（日本語対応） |

### ステータス表示
| 項目 | 仕様 |
|------|------|
| **標準表示** | HP/MP は数値表示を主とする。レトロ感を保つため、常時大きなゲージは出さない |
| **補助表示** | 危険時のみ色変化、点滅、小さな残量マーカーで状態を補足する |
| **オプション** | 簡易HP/MPバーは設定でONにできる |
| **経験値表示** | レベルアップ直前のみ小さな補助ゲージを許可。常時表示はしない |

### フォント仕様
| 用途 | サイズ | 備考 |
|------|--------|------|
| **本文** | 8×8px | 日英両対応、等幅 |
| **数値** | 8×8px | ダメージ表示用、太字バリエーション |
| **タイトル/見出し** | 16×16px | 大きめ表示用 |
| **小文字** | 4×8px（要検討） | 補助情報用 |

## 7.5 現代UX要素（GBCにはなかったもの）

### 追加するモダン機能
| 機能 | 説明 | 理由 |
|------|------|------|
| **オートセーブ** | マップ移動・バトル終了時に自動保存 | 事故によるデータロスト防止 |
| **速度倍率** | 1x/2x/4x切替 | 移動・バトルテンポ調整 |
| **ソート・フィルター** | モンスター一覧のソート/絞り込み | 400体管理の利便性 |
| **配合プレビュー** | 既知レシピや大まかな傾向のみ表示。未発見レシピは答えを出しすぎない | 発見の喜びを殺さず利便性を確保 |
| **最近の行動ログ** | 直近のバトル結果・入手モンスター | プレイ再開時の思い出し |
| **クエストログ** | 現在の目標・進行状況 | 迷子防止 |
| **簡易マップ** | 探索済み範囲のみ表示、設定でON/OFF | 探索の利便性と未知性の両立 |
| **お気に入りロック** | モンスターの誤配合防止 | 大切なモンスターの保護 |
| **図鑑検索** | テキスト検索、種族/属性フィルター | 400体からの検索性 |
| **バトルログ** | 戦闘中の行動履歴表示 | 戦略立案の参考 |

### 既定値の考え方
- 追加機能は便利でも、**初回起動時は当時感を壊しにくい設定** を既定値にする
- 配合の完全答え合わせ、常時ミニマップ、過剰な情報表示は標準ONにしない
- 「困ったら開ける」補助として提供し、最初から解法を並べない

### 追加しないもの（レトロ感を壊すため）
- 矢印ナビゲーション（次の目的地を示す矢印）
- ミッション達成度のパーセント表示
- リアルタイム通知
- SNS連携
- 実績ポップアップ

## 7.7 バトルUI

### レイアウト
```
┌─────────────────────────┐
│                         │
│    敵モンスター表示       │
│    (3体横並び)           │
│                         │
├─────────────────────────┤
│ 味方1 HP 123 / MP 45    │
│ 味方2 HP 087 / MP 32    │
│ 味方3 HP 201 / MP 18    │
├─────────────────────────┤
│ コマンドウィンドウ        │
│ ▶ たたかう               │
│   さくせん               │
│   どうぐ                 │
│   にげる                 │
└─────────────────────────┘
```

### バトルUI原則
- 1ターンの入力量を減らし、**準備と作戦の差** が前面に出る構成にする
- 味方ステータスは数値中心にし、色変化や点滅で危険状態を伝える
- `さくせん` から個体ごとの AI 方針と `直接指示` を切り替える
- 補助説明は画面下部のメッセージ領域で返し、サブウィンドウを乱立させない

### バトル演出
| 演出 | 仕様 |
|------|------|
| **攻撃** | スプライトが前方に微移動 + 被弾フラッシュ |
| **魔法/特技** | 画面全体にエフェクトオーバーレイ |
| **会心の一撃** | 画面シェイク + 専用SE |
| **撃破** | スプライト消滅アニメーション |
| **レベルアップ** | 専用ジングル + ステータス上昇表示 |

## 7.8 配合UI

### 配合画面フロー
```
親1選択 → 親2選択 → プレビュー表示 → 確認 → 配合実行 → 結果表示
```

### プレビュー表示内容
- 既知レシピなら生まれるモンスターの名前・種族・属性
- 未発見レシピなら結果候補の系統、ランク帯、傾向ヒントのみ
- 予想ステータス範囲
- 継承可能なスキルツリー一覧
- 変異の可能性表示（「まれに変異する可能性があります」）
- **警告**: お気に入りロック中のモンスターを親にしようとした場合

### スキルツリー継承選択
- 親1のスキルツリー一覧 + 親2のスキルツリー一覧を表示
- 最大3つまで選択
- 選択済みスキルの技一覧をプレビュー可能
