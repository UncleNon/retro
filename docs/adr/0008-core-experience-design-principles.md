# ADR-0008 Core Experience Design Principles

- Status: Accepted
- Date: 2026-03-15

## Context

初期携帯機モンスターRPGの魅力は、単純な内容量ではなく、限られた画面・限られた所持数・短い戦闘・重い配合判断の中で「自分だけの個体を作っている」と感じさせる設計にある。この企画でも、DQM系の配合中毒性と、初期携帯機らしい情報密度・メニュー速度を、抽象原則と具体ルールに分けて保存する必要がある。

## Decision

- コア体験は **unique-my-monster** と定義する
- 設計の第一原則は **不可逆性と救済の同居** とする
- 戦闘は短く、勝敗は戦闘前の準備、作戦、育成、配合で決まる構造を優先する
- 確率は捕獲、変異、ドロップなど局所的な場面に閉じ込め、長期目標は計画可能な決定論で支える
- UI規格は **160×144 / 20×18タイル / 8×8px** を基準とし、整数スケーリングを前提にする
- バトル文法は **短い4コマンド + 作戦AI** を基準にし、詳細介入は例外扱いにする
- 配合支援は許容するが、未発見レシピの答えを標準で見せすぎない
- 便利機能は許容するが、**デフォルトは当時感、オプションで現代快適** を守る
- DQMやポケモンから学ぶのは原理であって、固有表現は継承しない

## Rationale

- 配合の親消滅、所持制限、短い戦闘、説明帯つきメニューは、全て「判断の重み」と「高速な再試行」を両立させるために機能していた
- レトロ体験の本質は不便さそのものではなく、制約が生む選択圧と情報の圧縮にある
- 現代端末では快適機能を足せるが、標準体験を快適化しすぎると、元の設計思想が消える

## Consequences

- 配合、所持制限、作戦AI、UI説明帯、テキスト速度、オプション設計はこの原則に従って評価する
- 数値中心のHP/MP表示、4コマンド戦闘、未発見レシピへの情報制限もこの原則の評価対象に含める
- Vertical Slice では、20枠前後の携行制限、短いバトル、説明帯つきリスト、配合の不可逆性が実際に機能するかを検証する
- 参考作品の表現を写すのではなく、「なぜ気持ち良かったか」を自作のルールに翻訳する
