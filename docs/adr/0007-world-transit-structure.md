# ADR-0007 World Transit Structure

- Status: Accepted
- Date: 2026-03-15

## Context

20以上の世界を扱う本作では、世界間移動の体験が単なる一覧メニューや無機質なポータルだと、発見の喜びと世界観の不気味さが弱くなる。DQM系の「旅の扉」的ワクワクは参照しつつ、そのままなぞるのではなく、モンスター存在論と禁忌に接続した独自の移動構造が必要だった。また、建物の意味を「観測施設」「制御装置」と言い切ると世界の奥行きが安くなるため、意図不明のまま匂わせる余地が必要だった。

## Decision

- 世界間移動の基盤は **中央ハブ + 生きた門** とする
- 中央ハブの仮称は **塔** とする
- 塔は、聖堂、修道院、寺社、天文台、墓所のどれとも断定しない
- 塔は抽象的なロビー空間ではなく、通常の世界地形の中に存在する特異建築として扱う
- 各世界への入口は、巨大モンスターの遺骸や器官を思わせる「生きた門」とする
- 門の解放条件は、ストーリー進行、キーアイテム、トーナメントランク、モンスター系統との共鳴などを組み合わせる
- 世界移動は単純なメニュー遷移ではなく、「門に触れる」「条件に反応する」「越境する」演出を伴う
- 上位存在が生命や世界をこね回した痕跡に見えるよう設計するが、説明しすぎない

## Rationale

- 20+世界でも導線管理しやすく、ソロ開発でも構造を保ちやすい
- DQM的なワクワクは残しつつ、独自の不気味さと記号性を持たせられる
- モンスターの起源、変異種、配合、世界の分裂を一本の設定で接続しやすい
- 新世界の解放が、単なるリスト追加ではなく、物語と報酬の両方として機能する
- 目的が説明されすぎない建物にすることで、神話、宗教、宇宙、禁忌のどれにも寄せられる余白を確保できる

## Consequences

- `docs/requirements/03_world_and_story.md` と `docs/requirements/05_map_and_worlds.md` をこの構造に合わせる
- UI上の世界移動画面は、ワールドマップ一般論ではなく、塔からの遷移演出を前提に設計する
- 各世界には固有の門演出、門説明文、解放条件メタデータが必要になる
- 深層設定では「なぜ門が生きているのか」「なぜモンスター系統と共鳴するのか」を回収する必要がある
- ただし初期段階では「観測装置」「制御施設」と断定する表現を避け、匂わせに留める
