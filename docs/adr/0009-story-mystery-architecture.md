# ADR-0009 Story Mystery Architecture

- Status: Accepted
- Date: 2026-03-15

## Context

この企画では、ダークファンタジーの雰囲気だけでなく、村の生活、塔の禁忌、失踪事件、配合の倫理、世界の分裂が一本で繋がっている必要がある。また、プレイヤーが自分で気づいた感覚を持てるよう、伏線は一箇所に集約せず分散配置する必要がある。

## Decision

- 物語の導入は `小さな村 + 家畜番の主人公 + 昔から禁忌とされる塔` を基盤にする
- 主人公は事件に巻き込まれるのではなく、自分の意思で塔へ向かう
- 村の失踪事件には **書き換えられた家系・名前・所属** という共通項を持たせる
- 失踪は現在進行形ではなく、昔の出来事が共同体の記憶として残っている形にする
- 本編では `人間側が何を隠し、何を利用していたか` まで決着させる
- 深層では `なぜ世界がその構造を持つのか` を残し、エンドコンテンツで掘る
- 伏線は 50+ の断片として、図鑑、台詞、記録、建築、制度、配合へ散らす

## Rationale

- 共同体の生活と超常の接点にストーリーの手触りが出る
- 村人の失踪を個人因縁ではなく共同体の禁忌にすることで、世界の生々しさが増す
- 名前と所属の揺らぎを軸にすると、家系、養子、相続、配合、モンスター存在論を一つのテーマで束ねられる
- プレイヤーが“答えを聞かされた”のではなく“断片を繋げた”と思える構造が、この企画の理想に合う

## Consequences

- `docs/requirements/03_world_and_story.md` では導入、共通項、主筋、勢力、伏線バンクを明文化する
- 配合、図鑑、NPC会話、トーナメント、塔内部の演出も、この物語構造の一部として扱う
- 以後の世界詳細設計では、各世界ごとに `失踪 / 改名 / 所属の揺らぎ / 門への禁忌` の変奏を必ず持たせる
