## 1. 目的

- `確率は局所、長期目標は決定論` を運用可能なルールにする
- どこで揺らぎを許し、どこで絶対に許さないかを固定する
- レトロな手触りを保ちながら、現代プレイヤーが理不尽で離脱しない確率帯を持つ

---

## 2. 上位原則

### 2.1 許す揺らぎ

- 戦闘1ターン単位の小さなブレ
- エンカウントの間隔
- 勧誘やドロップの成否
- 状態異常のターン長
- mutation の発生

### 2.2 禁止する揺らぎ

- ストーリーの分岐成立条件
- 特殊配合の子種判定
- レベルアップ時の基本成長結果
- ボス撃破後の主要解放
- clue の本編回収可否

### 2.3 体験ルール

| 原則 | 内容 |
|------|------|
| `short-term uncertainty` | 1戦、1探索、1勧誘は揺れてよい |
| `mid-term recoverability` | 数回試せば収束する |
| `long-term planability` | 目標育成は表に見えなくても内部では計画可能 |

## 3. 許可対象一覧

| 対象 | 許可 | 備考 |
|------|------|------|
| 物理ダメージ乱数 | 〇 | 小幅のみ |
| 呪文ダメージ乱数 | 〇 | 小幅のみ |
| クリティカル | 〇 | 明確な低率 |
| 命中 / 回避 | 〇 | 高すぎる外しを避ける |
| 状態異常成功 | 〇 | `int` と `res` で補正 |
| 状態異常継続ターン | 〇 | 1〜2ターン幅中心 |
| エンカウント歩数 | 〇 | min/max 方式 |
| 勧誘成功 | 〇 | 手札で改善可能 |
| ドロップ率 | 〇 | common は高め |
| mutation 発生 | 〇 | 低率だが触媒で押せる |
| 配合結果の種 | × | 決定論 |
| 配合継承候補の母集合 | × | 決定論 |
| main story 解放 | × | フラグ依存のみ |
| gate 開放 | × | 条件達成で必ず解放 |

## 4. Probability Budgets

### 4.1 戦闘

| 要素 | 目安 |
|------|------|
| 物理ダメージ幅 | `±15%` 以内 |
| 呪文ダメージ幅 | `±10%` 以内 |
| 通常クリティカル | `2%〜4%` |
| 高クリ trait 持ち | `5%〜9%` |
| 通常命中率下限 | `70%` を切らせない |
| 補助命中率上限 | `95%` |

### 4.2 状態異常継続

| 状態 | 継続ターン |
|------|------------|
| 眠り | 1〜3 |
| 混乱 | 1〜3 |
| 封印 | 2〜4 |
| 速度低下 | 2〜4 |
| mark / hush 系 | 2〜3 |

### 4.3 エンカウント

| map tier | `min_step` | `max_step` |
|----------|-----------:|-----------:|
| 平穏地帯 | 7 | 14 |
| 通常野外 | 5 | 11 |
| 危険野外 | 4 | 9 |
| ダンジョン深部 | 3 | 8 |

### 4.4 勧誘

| 条件 | 目標体感 |
|------|----------|
| 何もしない | あまり入らない |
| HP減少のみ | たまに入る |
| HP減少 + 状態異常 | 手応えが出る |
| HP減少 + 状態異常 + 適正bait | 狙える |
| 上位bait 連打 | 90%保証までは行かない |

### 4.5 mutation

| 条件 | 発生率目安 |
|------|-----------:|
| 通常配合 | `1.5%` |
| 高世代 | `+0.5%` |
| 高Lv合計 | `+0.5%` |
| 触媒あり | `+2% 〜 +6%` |
| 特定世界補正 | `+1% 〜 +3%` |

Hard cap:

- 初回リリースでは単回 mutation 率を `12%` 超にしない

## 5. 救済ポリシー

### 5.1 Recruit Pity

- 明示しない
- 同種同ランク帯で `連続失敗5回` を超えた場合、内部補正 `+5`
- `連続失敗8回` を超えた場合、内部補正 `+10`
- rare / upper-rank では補正上限を半分にする

### 5.2 Drop Recovery

- 必須進行素材はランダムドロップにしない
- 準必須の crafting 素材は `first_clear_bonus` で1個保証
- clue 系アイテムは `drop` でなく `object pickup` に寄せる

### 5.3 Encounter Fairness

- `max_step` 到達時は必ず遭遇
- 一本道強制マップでは `min_step` を高めに置く
- 再探索後半は repel 系で圧縮可能

## 6. Dungeon Randomness

### 6.1 許すもの

- 部屋の接続順
- 宝箱位置の候補差し替え
- 回復小部屋の出現有無
- 群れ配置の差

### 6.2 禁止するもの

- メイン導線が消える構造
- 伏線イベント部屋の未生成
- ボス到達不能
- 重要ギミックの解法自体がランダム

---

## 7. AI Randomness

### 7.1 許すもの

- 同点行動候補の tie-break
- 条件一致時の攻撃 / 補助の揺らぎ
- 群れ行動の狙い分散

### 7.2 禁止するもの

- ボスが勝ち筋を捨てるほどの無作為
- 性格や作戦が読めなくなるレベルのブレ
- 一戦ごとに人格が変わったように見える挙動

---

## 8. Visible Communication

- 数値確率は表示しない
- 代わりに以下で手応えを出す

| 要素 | 伝え方 |
|------|--------|
| 勧誘 | 敵の反応文、体勢、アイコン色 |
| 状態異常 | 効きそう / 効かなそうな短文 |
| エンカウント | 地形、音、風、草丈 |
| mutation | 配合施設の反応、触媒の説明文 |

---

## 9. QA Checklist

- その確率は「ノートを取りたくなる」側か、「運ゲー」に倒れていないか
- 数回の試行でプレイヤーが学習可能か
- 本編進行が RNG に依存していないか
- 救済が表に見えすぎてレトロ感を壊していないか
