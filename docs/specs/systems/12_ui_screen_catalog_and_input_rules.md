## 1. 目的

- master_design_matrix で `open` / `next` のまま残っているUI項目を全て固定する
- 全画面の寸法、情報密度、遷移、入力を実装可能な粒度で定義する

## 2. グローバル規格

### 2.1 画面

| 項目 | 値 |
|------|----|
| 内部解像度 | 160×144 px |
| タイルグリッド | 20×18 tiles |
| 1タイル | 8×8 px |
| 描画スケール | 整数倍のみ |
| aspect ratio | keep（レターボックス許可） |

### 2.2 テキスト

| 項目 | 値 |
|------|----|
| フォント行高 | 8 px |
| 文字実体 | 6×8 px（固定幅） |
| 日本語文字幅 | 全角 = 8px、半角 = 4px |
| 1行最大文字数 | 全角18文字 / 半角36文字 |
| 説明帯 | 画面下部2行 = 16px |

### 2.3 テキスト速度

| 設定 | 文字/フレーム | 備考 |
|------|-------------:|------|
| おそい | 1 | 約16文字/秒 |
| ふつう（既定） | 2 | 約32文字/秒 |
| はやい | 4 | 約64文字/秒 |
| いちばんはやい | 全文即表示 | Aボタンで即表示 |

### 2.4 カーソル

| 項目 | 値 |
|------|----|
| サイズ | 8×8 px |
| アニメーション | 2F、16Fサイクル（約0.27秒で点滅） |
| 移動方式 | ジャンプ（スライドなし） |
| 色 | パレット内の最明色 |

### 2.5 ウィンドウ枠

| 項目 | 値 |
|------|----|
| 構成 | 9-slice（4角 + 4辺 + 1塗り = 9タイル） |
| 枠太さ | 1px線 |
| 内側余白 | 2px（上下左右均一） |
| 角タイル | 8×8 px |
| 背景タイル | 単色塗り（最暗色の1段明るい色） |

## 3. 入力体系

### 3.1 論理ボタン

| ボタン | 物理マッピング（コントローラ） | タッチ | 役割 |
|--------|-------------------------------|--------|------|
| A | 決定ボタン | タップ | 決定、実行、会話送り |
| B | キャンセルボタン | 左スワイプ or 戻るアイコン | キャンセル、1階層戻り |
| START | メニューボタン | メニューアイコン | ルートメニュー開閉 |
| SELECT | 整理ボタン | 長押し | ソート、並び替え |
| D-PAD | 十字キー / スティック | 仮想パッド | 4方向移動、リスト移動 |

### 3.2 iPhone 仮想パッド

| 項目 | 値 |
|------|----|
| パッド位置 | 画面左下 |
| パッドサイズ | 画面短辺の 28% を直径とする |
| ボタン位置 | 画面右下 |
| ボタンサイズ | 48×48 dp（Retina基準） |
| デッドゾーン | パッド中心から 12% 以内 |
| 半透明度 | 40%（操作中は 70%に上昇） |
| 片手モード | v1 では非対応。将来検討 |

### 3.3 操作の一貫性ルール

- B は **常に** 1階層戻る。例外なし
- A は **常に** 決定 or 会話送り。例外なし
- メニューの最大深度は **4階層** まで
- どの画面からも、B を押し続ければ必ずフィールドに戻れる

### 4.6 モンスター詳細（ページ送り）

ページ1: ステータス
- 名前、Lv、種族名、系統アイコン
- HP/MP/ATK/DEF/SPD/INT/RES（数値のみ）
- 性格、忠誠度文言

ページ2: スキル
- 習得済みスキル一覧（最大8）
- 各スキルの説明帯表示

ページ3: 系譜
- 親A/親Bの種族名（不明なら「？」）
- plus_value
- 世代数

ページ4: 図鑑文
- 図鑑テキスト（2-3行）

### 4.7 どうぐ画面

```
┌─────────────────────────┐
│ どうぐ         12/20      │
├─────────────────────────┤
│ ▶ひからび草     ×3       │
│   透き水        ×2       │
│   乾き餌        ×5       │
│   骨縄          ×1       │
│   印粉札        ×2       │
│   ...                    │
├─────────────────────────┤
│ HPを20回復する            │
│ フィールド / バトル       │
└─────────────────────────┘
```

- 説明帯は **選択中 item と同期** し、少なくとも `効果1行 + provenance / 用途の短文1〜2行` を返す
- provenance の短文 source は `item_text_master.csv` の `menu_strip` を正とし、未登録 item のみ `item_master.description` へ fallback してよい
- key item は一覧で `[K]` を付けつつ、説明帯では固有名を崩さず `大事なもの` として扱う
- 施設 preview / buy confirm の売り手文は `item_text_master.csv` の `shop_voice` を正とし、lookup は `shop_id -> scope_id -> description fallback` の順で解決する

### 4.9 図鑑画面

| 要素 | 仕様 |
|------|------|
| 一覧表示 | 系統別タブ + 番号順 |
| 未発見 | シルエット + ??? |
| 発見済み未所持 | シルエット + 名前 |
| 所持/配合済み | スプライト + 名前 + 図鑑文 |
| フィルタ | 系統 / ランク / 発見状態 |
| ソート | 番号 / ランク / 系統 |

- 一覧カーソル移動中の説明帯は、選択中 monster の `family / rank / short lore` を返す

## 5. ダンジョン簡易マップ

| 項目 | 値 |
|------|----|
| 表示モード | 設定でON/OFFを切り替え。既定 OFF |
| 表示位置 | 画面右上 4×4 tiles |
| 内容 | 探索済み部屋のみ。未探索は非表示 |
| アイコン | 現在地点 = 点滅ドット、階段 = ▲/▼、宝箱 = □ |

## 6. 配合履歴の見せ方

| 項目 | 値 |
|------|----|
| 形式 | テキストログ（系譜ツリーは endgame 解放） |
| 表示 | 「きろく」メニュー内の「はいごうれきし」 |
| 1件の情報 | 日時、親A名/種、親B名/種、子名/種、plus_value |
| 保持件数 | 最新50件 |
| 系譜ツリー表示 | endgame で「けいふず」コマンド解放 |

## 7. 町/村UIの案内密度

| 要素 | 密度 |
|------|------|
| 看板 | 施設名のみ。説明文は入れない |
| 施設アイコン | 入口タイルに1タイルのシンボル（道具屋=壺、宿=星、牧場=柵） |
| NPC誘導 | 初回訪問時のみ、最寄りNPCが1文で案内 |
| ミニマップ | 町では非表示 |

## 8. QA Checklist

- 全画面が 20×18 tiles に収まるか
- テキストが全角18文字で折り返さずに収まるか
- B連打でフィールドまで確実に戻れるか
- 説明帯がリスト選択と同期して更新されるか
- 仮想パッドがゲーム画面を邪魔しないか
- 配合の2回確認が確実に動作するか
- 未発見モンスターのシルエットが IP 侵害にならないか
