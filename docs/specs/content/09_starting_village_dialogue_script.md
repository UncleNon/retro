## 1. 凡例

### 1.1 フェーズ定義

| Phase | トリガ | 状態 |
|-------|--------|------|
| `P0` | `EVT_VIL_000_START` | 塔に入る前。朝の村 |
| `P1` | `EVT_W01_090_RETURN_FROM_FIRST_WORLD` | W-001 帰還直後 |
| `P2` | 中盤帰還 (W-003 以降) | 沈黙に亀裂が入り始める |
| `P3` | 終盤帰還 (W-005 以降) | 告白 or 深化した沈黙 |

### 1.2 テキストボックス仕様

- 全角 18 文字 x 最大 3 行 = 1 テキストボックス
- `「」` 内が台詞、`＊` で始まる行は動作・演出ト書き
- `>>>` は次のテキストボックスへの送り
- 各台詞 ID は `DLG-{NPC番号}-{Phase}-{連番}` 形式

### 1.3 文体規則 (05 準拠)

- 村人: 生活語、代名詞で濁す、名指しを避ける
- 長老: 言い切らない、時間語 (代々、昔から) 多用
- 記録番: 受動態、制度語 (転出、整理、再記載)
- 子ども: 大人の隠語を意味抜きで復唱
- 塔を直接呼ばない。「上のほう」「あっち」「北の奥」

- [x] P0 で最低 3 台詞、P3 で最低 1 台詞
- [x] 塔を直接呼ぶ台詞がない (「北のほう」「あっち」「上のほう」「北の奥」「あの建物」で代替)
- [x] 長老は時間語 (代々、昔から、ならわし) を使用
- [x] 記録番は受動態・制度語 (転出、整理、記載、管轄) を使用
- [x] 子どもは大人の隠語を意味なく復唱 (「数え直す」「数に入れなくていい」)
- [x] 主人公は無言。NPC がリアクションする構造
- [x] 1 テキストボックス = 全角 18 字 x 3 行以内
- [x] 村人は名指しを避け、「どこの家」「あの人」で濁す
- [x] P3 で全員が全てを説明する構造になっていない
- [x] 単独黒幕がいない。責任は分散している
- [x] 失踪件数 9 件が一貫している
- [x] 主人公 45 歳に対する「分かっている相手への誤魔化し」の口調
- [x] 「fear / gain / tenderness / shame」の 2 軸以上が台詞に反映
- [x] 記録・墓・布・歌・柵・灰・紐・名の 8 系統で失踪の残滓が表出

### 22.3 Phase 別の設計意図

| Phase | 空気 | 台詞の機能 |
|-------|------|-----------|
| P0 | 静かな日常。違和感は断片的 | 生活感の確立、伏線の種まき |
| P1 | 動揺と封じ込め | 村人が主人公の帰還に反応し、壁を厚くする |
| P2 | 亀裂の露出 | 隠していたことが言葉の端から漏れ始める |
| P3 | 告白か深化した沈黙 | 全てを語らないが、自分の加担を認め始める |
