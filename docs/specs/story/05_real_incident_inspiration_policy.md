# 05. Real Incident Inspiration Policy

> **ステータス**: Draft v1.0
> **最終更新**: 2026-03-15
> **役割**: 実在の神隠し譚、失踪事件、未解決事件、共同体伝承を創作の骨格として参照する際の抽象化ルール

---

## 1. 目的

- 実在の出来事の `手触り` と `不可解さ` を参考にしつつ、露骨な引用や消費にしない
- 「何が起きたか」よりも、「どう語り継がれ、どう歪み、どう隠されたか」を設計へ取り込む
- ミステリーの深みを、安易な残酷描写ではなく、共同体と記録の層で作る

---

## 2. 参照してよいもの

| 参照対象 | 使い方 |
|----------|--------|
| 神隠し、境界伝承 | 山、森、橋、辻、祭場などの境界感覚 |
| 未解決失踪 | 記録不一致、証言の食い違い、最後の目撃情報 |
| 共同体が抱える噂 | 語る者ごとに細部が違う伝承構造 |
| 行政的処理 | 行方不明、死亡、転出などの分類のズレ |
| 宗教儀礼 | 禁忌の生活化、言い換え、手順化 |

---

## 3. 参照してはいけないもの

- 特定事件の固有名詞、場所、年齢構成、家族構成をほぼそのまま使う
- 実在被害者の痛ましさを演出の燃料にする
- 現代に近い未解決事件を、そのままファンタジー化して消費する
- 現実の差別や被害を、配慮なく「味」として置く

---

## 4. 抽象化の単位

### 4.1 直接引用しない

使うのは `事件の骨格` だけにする。

| 骨格 | 変換例 |
|------|--------|
| 最後の目撃が曖昧 | 門前で名前だけ記録に残る |
| 証言が一致しない | 世界ごとに禁忌の言い換えが違う |
| 記録上の扱いが食い違う | 死亡 / 転出 / 失踪の書換え |
| 場所に伝承が付着する | 塔、祠、墓、街道の禁忌 |
| 時間が経つほど神話化する | 村の迷信と制度の接着 |

### 4.2 抽象化の優先順位

1. 共同体がどう受け止めたか
2. 記録がどう歪んだか
3. 境界空間がどう意味づけられたか
4. 語り手ごとに何が変わるか
5. 事件そのものの異常性

---

## 5. 本作に取り込む構造

| 構造 | 本作での使い方 |
|------|----------------|
| `語り継がれるうちに理由だけが残る` | 「塔に近づくな」が行動規範として残る |
| `関係者ほど断片しか言わない` | 長老、記録官、宗教者の会話温度差 |
| `行政分類が真実を隠す` | 失踪の死亡 / 転出処理 |
| `共同体が犠牲を生活の継続で正当化する` | 家督、通行権、闘技、配合文化 |
| `場所が噂の器になる` | 塔、門、墓苑、湿地、鏡森 |

---

## 6. narrative filters

### 6.1 採用前に通す質問

1. この要素は固有事件の焼き直しになっていないか
2. 何を抽象化したのか一文で説明できるか
3. 残酷さではなく、不可解さと制度性で効いているか
4. ストーリー全体の論理と接続しているか

### 6.2 採用後に通す質問

1. 語りが被害者消費に寄っていないか
2. その世界や共同体の論理があるか
3. ただ怖いだけで終わらず、後で意味が反転するか

---

## 7. QA Checklist

- 「実在っぽさ」が、固有の事件参照でなく構造から出ているか
- 伝承、行政、生活、宗教のどれか二つ以上に跨っているか
- 本編の `共同体の沈黙` テーマに接続しているか
