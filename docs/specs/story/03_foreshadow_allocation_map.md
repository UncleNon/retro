## 1. 目的

- 72件の違和感を `どこに / どう置き / いつ回収するか` まで管理する
- 伏線を会話だけに寄せず、建築、図鑑、戦闘、道具、門反応へ分散する
- 無言主人公でもプレイヤーの認識が段階的に深まる構造を作る

---

## 2. reveal tier

この `T1〜T5` は clue 単位の粒度であり、`story/01_story_bible.md` や
`story/04_main_story_beats_and_world_sequence.md` の物語段階そのものとは 1:1 で対応しない。

| tier | 説明 |
|------|------|
| `T1` | 単なる違和感 |
| `T2` | 反復により規則が見え始める |
| `T3` | 名 / 家系 / 所属 / 門 に接続する |
| `T4` | 制度や責任主体に繋がる |
| `T5` | 本編内で意味が固定される |

## 4. 運用ルール

- `seen` は missable でも `resolved` は missable にしない
- 1世界で真相を2つ以上確定させない。ただし `terminal_world` では、同一決着線上の複数 clue を束ねてよい
- 同一 clue を同一媒体で繰り返しすぎない
- 主要真相は最低3媒体で冗長化する

---

## 5. QA Checklist

- clue が塔だけに偏っていないか
- 制度、共同体、モンスター、建築にも分散しているか
- 序盤で語彙を出しすぎていないか
- 本編完結に必要な clue の回収導線が保証されているか
