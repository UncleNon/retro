## 1. 目的

- 40〜60曲のBGMと40〜60種のSEを、一貫した音楽言語で管理する
- AI生成→手修正のワークフローを、視覚アセットと同じ品質管理水準で運用する
- 世界ごとの空気、禁忌の温度、生活感を音で伝える

---

## 2. 音の思想

### 2.1 キーワード

- 牧歌的だが影がある
- 素朴だが洗練されている
- GBCの4chで成立する密度感を超えない
- 沈黙も音の一部

### 2.2 やってはいけないこと

- フルオーケストラ風の豪華アレンジ
- 現代EDM的なドロップやビルドアップ
- メロディの詰め込みすぎ（1曲にテーマは2つまで）
- 感情を押し付ける大仰な表現
- 効果音の過剰な派手さ

## 3. 技術仕様

### 3.1 BGM

| 項目 | 仕様 |
|------|------|
| 音源スタイル | 8bit GBC PSG風チップチューン |
| チャンネル構成 | 4ch基本（矩形波×2 + 波形メモリ×1 + ノイズ×1） |
| 拡張 | 必要に応じて5-6chまで許可（ただしGBC感を壊さない） |
| フォーマット | OGG Vorbis（Godot標準、ストリーミング再生） |
| サンプルレート | 44100Hz |
| ビット深度 | 16bit |
| ループ | シームレスループ必須。ループポイントをメタデータで管理 |
| クロスフェード | 場面転換時 0.3〜0.8秒。フェードアウト→短い無音→フェードイン |
| ラウドネス | -14 LUFS ±1 で正規化（全曲統一） |
| 長さ | 30秒〜120秒のループ単位 |

### 3.2 SE

| 項目 | 仕様 |
|------|------|
| 音源スタイル | 8bit風。BGMと調和する |
| フォーマット | WAV（即時再生、非ストリーミング） |
| サンプルレート | 44100Hz |
| ビット深度 | 16bit |
| 長さ | 0.05〜2.0秒 |
| ラウドネス | ピーク -6 dBFS 以下 |
| ランダムピッチ | 再生時 ±8%（0.92〜1.08倍）で機械的反復を避ける |
| 同時再生上限 | SE 4音まで（衝突時は優先度で破棄） |

### 3.3 ジングル

| 項目 | 仕様 |
|------|------|
| フォーマット | OGG |
| 長さ | 1.5〜7秒 |
| BGMとの関係 | ジングル再生中はBGMを duck（-12dB） |
| ラウドネス | BGMと同じ -14 LUFS |

### 4.3 BGM切り替えルール

| 遷移 | 方法 | 時間 |
|------|------|------|
| フィールド→バトル | フェードアウト 0.3s → 戦闘BGM即再生 | 0.3s |
| バトル→フィールド | 戦闘BGM停止 → フィールドBGM頭から | 0.5s |
| 勝利ジングル→フィールド | ジングル完了 → フェードイン 0.8s | ジングル長+0.8s |
| 拠点→ダンジョン | クロスフェード 0.5s | 0.5s |
| ダンジョン→ボス部屋 | 一瞬の無音 0.2s → ボスBGM | 0.2s |
| 門越え | BGM完全フェードアウト 1.0s → 無音 1.5s → 新世界BGM | 2.5s |

| 優先度 | カテゴリ |
|-------:|----------|
| 1（最高） | ジングル、ゲームオーバー |

| 6（最低） | アンビエントレイヤー |

## 8. 塔と門の音の設計

### 8.1 塔の音

塔は「音が減る場所」として設計する。

| 塔の深度 | 音の変化 |
|----------|----------|
| 塔前 | 村のアンビエントが薄くなる。鳥と家畜が消える |
| 塔入口 | BGMのテンポが落ちる。環境音は反響のみ |
| 塔中層 | BGMが断片化する。メロディが途切れる |
| 塔深層 | BGMはドローンのみ。足音と反響が目立つ |
| 門の前 | ほぼ無音。低周波の圧だけが残る |

### 8.2 門の音

- 門は「音が歪む場所」
- 通過の瞬間にピッチが微妙にずれる（±15cent）
- 門の向こう側に着いた瞬間、新世界のアンビエントが0.5秒だけ異常に大きく聞こえ、すぐ正常化する
- これにより「越境した」感覚を音で演出する

## 9. AI生成プロンプト方針

### 9.1 BGM生成

```text
[必須要素]
- ジャンル: "8-bit chiptune", "Game Boy Color PSG style"
- チャンネル: "4 channels only" or "primarily 4 channels"
- ムード: カテゴリ表の「感情」列を使う
- テンポ: カテゴリ表のBPM範囲を指定
- ループ: "seamless loop, approximately {length} seconds"
- 禁止: "no modern reverb, no orchestral instruments, no vocals, no dubstep drops"

[世界別の追加指示]
- 牧畜圏: "pastoral, pentatonic scale, gentle, with subtle unease"
- 海路圏: "arpeggiated waves, lighthouse bell, trade winds"
- 弔い圏: "bell tones, slow, deliberate pauses, mourning"
- 帳法院圏: "mechanical, typewriter rhythm, cold precision"
```

### 9.2 SE生成

```text
[必須要素]
- スタイル: "8-bit sound effect, retro game style"
- 長さ: "{length} seconds"
- 用途: SEの説明
- 禁止: "no realistic sounds, no 16-bit or higher fidelity, keep it simple and crisp"
```

### 9.3 品質管理チェックリスト

- [ ] ループポイントに不自然な切れ目がないか
- [ ] 音量が -14 LUFS ±1 に収まっているか
- [ ] BGMとSEを同時再生してSEが埋もれないか
- [ ] 既存楽曲との類似度チェック（Shazam等で確認）
- [ ] GBC 4ch感が崩れていないか
- [ ] 門反応SEがBGMの調性と衝突しないか
- [ ] 長時間ループで耳が疲れないか（3分以上連続試聴テスト）
