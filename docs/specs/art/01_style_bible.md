## 1. 目的

- AI生成と手修正を跨いでも、絵の人格が崩れないようにする
- 400体 / 20+世界 / UI / タイルセットを **同じ美術言語** で束ねる
- 「GBC風」を表層の懐古で終わらせず、制約から逆算した設計ルールとして運用する

---

## 2. ビジュアル思想

### キーワード

- 牧歌
- 冷たい異物
- 静かな禁忌
- 生活の染み
- 神話の断片
- 歪んだ素朴さ

### 基本方針

- 可愛さは許容するが、**完全な無垢さにはしない**
- 不気味さは露悪ではなく、**違和感として滲ませる**
- 塔や門の要素は、すべてを説明せず、生活圏に混じる異物として置く
- UIもフィールドも、**角の丸い安心感** と **冷たい直線の異物感** を併置する

---

## 3. ピクセル規格

| 項目 | ルール |
|------|--------|
| 内部解像度 | 160×144 |
| 基本グリッド | 20×18 tiles |
| タイル | 8×8 px |
| フィールドキャラ | 16×16 px |
| battle sprite | 24×24 / 32×32 / 48×48 / 56×56 |
| UI corner tile | 8×8 px |
| フォント行高 | 8 px |

### アウトライン

- 原則 1px
- 完全な黒 `#000000` 固定ではなく、**最終パレット上の最暗色** を使うことも許可
- ただし見た目上は「黒縁」と認識できる強度を保つ

### 禁止

- サブピクセル
- 自動アンチエイリアス
- 1px 未満の細線表現
- ガウシアンぼかし由来の曖昧な輪郭

---

## 4. パレット設計

### 画面全体の考え方

- GBCの制約を模倣しつつ、現代端末で破綻しないよう **運用上は 32色ベース** で管理する
- 1アセットで使う色数はさらに絞る

### アセット別の色数目安

| 対象 | 推奨色数 |
|------|---------:|
| 序盤モンスター | 4〜6 |
| 中盤モンスター | 5〜8 |
| 上位モンスター | 6〜10 |
| UIウィンドウ | 3〜4 |
| タイル1セット | 8〜16 |

### 色の役割

| 役割 | ルール |
|------|--------|
| 最暗色 | 外枠、影、穴、目 |
| 主色 | その対象の人格 |
| 補色 | 部位差、属性差、血統差 |
| 異物色 | 門、塔、禁忌、変異の印 |

### 世界別の色温度

| 世界種別 | 傾向 |
|----------|------|
| 村 / 生活圏 | 土、煤、乾草、曇りガラス、鈍い緑 |
| 塔周辺 | 灰青、白骨、冷たい金属、濁った月光 |
| 異界自然 | くすんだ鮮色、沈んだ補色 |
| 深層 / 門内部 | 反射白、冷紫、濁った金、血のような暗赤 |

## 5. モンスターシルエット法則

### family ごとの一次形

| 系統 | 一次形 | 禁止しがちな型 |
|------|--------|----------------|
| `beast` | 四肢の踏ん張り、体重感 | 単なる犬猫の丸写し |
| `bird` | 縦長、くちばしや首の流れ | generic コウモリ翼悪魔 |
| `plant` | 下重心、張る、垂れる、絡む | 花一輪だけの記号化 |
| `material` | 硬さ、角、組み合わせ | ただの道具に顔を付けただけ |
| `magic` | 反復模様、揺らぎ、非対称 | 漠然とした幽霊玉 |
| `undead` | 欠損、乾き、空洞 | グロだけで押す造形 |
| `dragon` | 骨格の説得力、尾・首の流れ | 既視感の強い西洋竜 |
| `divine` | 静けさ、儀式性、視線の強さ | 単なる天使風 |

### ランクごとの見た目圧

| Rank | ルール |
|------|--------|
| E | 一目で読める。輪郭は単純 |
| D | 読みやすさ優先、部位差は1つ増やす |
| C | シルエットに癖を1つ入れる |
| B | 非対称を許可、印象的な部位を持たせる |
| A | 威圧感、異物感、背景を匂わせる |
| S | “見たことのない骨格” を1箇所入れる |

## 6. モチーフの変形法則

### 現実モチーフ

- 現実の動物 / 植物 / 道具をそのまま使わない
- 必ず以下のいずれかを混ぜる
  - 生活の痕跡
  - 記録 / 名前 / 印
  - 葬送 / 儀式
  - 塔 / 門 / 継ぎ目

### 神話 / 古典モチーフ

- 有名固有名詞をそのまま引かない
- 使うのは以下の断片
  - 姿勢
  - 象徴物
  - 儀式的役割
  - 禁忌の構図

### 優先する歪め方

1. 生活道具が生き物化している
2. 生き物に制度や記録の痕が刻まれている
3. 家畜や村の景色が神話化している
4. 神話的存在が生活圏へ縮退している

## 7. UIデザインルール

### ウィンドウ

- 角は明確に 8×8 の部材感を持たせる
- 枠は厚すぎない
- 内側余白は 2px 〜 4px を守る
- 完全な豪華装飾より、**使い込まれた手触り** を優先

### カーソル

- 8×8 内で完結
- 2フレームで十分
- ちらつきすぎない

### 数値表示

- HP / MP は文字列主役
- 危険表示は `色 / 点滅 / 小マーカー` で補助
- “モダンUIの長いバー” を主役にしない

### 余白

| 対象 | ルール |
|------|--------|
| リスト項目間 | 0〜1px |
| ウィンドウ内余白 | 2〜4px |
| セクション分割 | 8px 単位 |

---

## 8. タイルセットルール

### 地面

- 8×8 単位で情報を詰め込みすぎない
- 1タイル単体で意味を持たせすぎず、**並びで読ませる**

### 建材

| 建材 | ルール |
|------|--------|
| 木 | 乾き、煤、割れ目を優先 |
| 石 | 平滑でなく、少し歪ませる |
| 骨 / 門材 | 有機物と建材の中間にする |
| 金属 | 鏡面にせず、冷たく鈍い反射 |

### 塔関連

- 完全左右対称を避ける
- 部分的な反復模様は許可
- 「宗教施設」「観測施設」「墓所」のどれとも断定できない混成感を保つ

---

## 9. アニメーション予算

| 対象 | 基本予算 |
|------|---------:|
| 主人公歩行 | 2F × 4方向 |
| 主要NPC歩行 | 2F × 4方向 |
| モブNPC | 1F or 2F |
| battle待機 | 2F |
| battle攻撃 | 2〜3F |
| hit | 2F |
| 門反応 | 4〜6F |

### 禁止

- ぬるぬるした中割り過多
- 変形しすぎる squash/stretch
- pixel cluster を壊す回転 / 拡縮

---

## 10. AI生成の禁止事項

- 既存IP名を直接プロンプトに入れない
- “pokemon-like”, “dragon quest style” のような寄せ指定をしない
- 過剰な装飾や背景込みで出さない
- 1体の中にモチーフを詰め込みすぎない

### 生成の順序

1. silhouette 案
2. motif の確認
3. palette 仮当て
4. pixel cleanup
5. animation 想定

---

## 11. レビュー観点

### 必須チェック

- 1x 表示で読めるか
- 系統内で被っていないか
- 村 / 塔 / 世界の文脈に接続しているか
- prompt から再生成可能か
- 既存IPの記憶に引っ張られすぎていないか

### fail 例

- ただのかわいい家畜
- ただの悪魔
- ただの神話生物
- 色だけ変えた既視感モンスター
- 解像度に対して情報過多
