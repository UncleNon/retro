## 1. 目的

- pixel-perfect な UI 実装で `文字は読めるが気持ちよくない` 状態を防ぐ
- フォント、ウィンドウ、説明帯、数値、カーソル、モバイル入力を同じ規格で量産する
- battle, menu, codex, dialog, shop の UI が別物に見えないようにする

---

## 2. 基本グリッド

| 項目 | 値 |
|------|----|
| 内部解像度 | `160x144 px` |
| UI 基本グリッド | `8x8 px` |
| 1画面 | `20x18 tiles` |
| 最小ウィンドウ角材 | `8x8 px` |
| 説明帯高さ | `16 px` |
| 最小選択行高 | `8 px` |
| 標準選択行高 | `10 px` まで許可 |

---

## 3. フォント規格

### 3.1 文字セル

| 種類 | セル | 実描画の目安 |
|------|------|--------------|
| 日本語全角 | `8x8` | `7x7` 以内 |
| 英数半角 | `8x8` | `5x7` or `6x7` |
| 記号 | `8x8` | `3x7` 〜 `6x7` |
| 数字等幅 | `8x8` | `5x7` 固定 |

### 3.2 行送り

- 標準行送りは `8 px`
- 説明帯 2 行も `8 px + 8 px`
- 強調行でも `9 px` を超えない

### 3.3 字間

| 用途 | 既定 |
|------|------|
| 本文 | `0 px` |
| 数値列 | `0 px`, 等幅前提 |
| UIラベル | `0 px` |
| 強調見出し | `0〜1 px` |

### 3.4 禁止

- 可読性のための自動スケーリング
- 可変幅で数値列が揺れる実装
- font ごとに line height を変えること

## 4. テキスト表示ルール

### 4.1 折り返し

- 句読点ぶら下がりはしない
- 語尾の 1 文字だけを次行に落としにくいよう、文節単位の調整を優先する
- 戦闘ログは 1 文を短くし、UI 側の無理な自動折返しに頼らない

### 4.2 文字数目安

| 領域 | 1行の目安 |
|------|-----------:|
| 説明帯 | 16〜18 全角 |
| 会話窓 | 16〜18 全角 |
| メニュー右窓 | 6〜8 全角 |
| 数値付きリスト | 10〜14 全角 |

### 4.3 既定のテキスト速度

| 設定 | 文字 / 秒の目安 |
|------|----------------:|
| `slow` | 18 |
| `normal` | 28 |
| `fast` | 40 |
| `instant` | すぐ表示 |

既定値は `normal` とする。

## 5. battle UI 規格

### 5.1 画面配分

| 領域 | 行範囲 | 用途 |
|------|--------|------|
| 上段 | `0-11` | 敵、味方数値、演出 |
| 下段 | `12-17` | メッセージ + コマンド |

### 5.2 味方数値ブロック

| 項目 | ルール |
|------|--------|
| 味方数 | 3 体分 |
| 表示 | `名前 / HP / MP` を1行ずつ |
| 数値桁 | HP 3桁, MP 3桁を想定 |
| 危険表示 | 文字色変化 or 小マーカー点滅 |

### 5.3 コマンド窓

| 項目 | 値 |
|------|----|
| 配置 | 下段右寄り |
| コマンド数 | 4 |
| 並び | 2x2 or 4縦 のどちらか |
| 既定 | `たたかう / さくせん / どうぐ / にげる` |

### 5.4 戦闘ログ

- 1回に出す文は最大 2 文
- 1 文の長さは 18 全角以下を推奨
- 追加情報は次メッセージへ逃がす

## 6. menu UI 規格

### 6.1 ルートメニュー

| 項目 | 値 |
|------|----|
| 幅 | `8〜9 tiles` |
| 高さ | `8〜11 tiles` |
| 位置 | 右寄せ |
| 背景 | フィールド上に重ねる |

### 6.2 リストウィンドウ

| 種別 | 推奨サイズ |
|------|------------|
| アイテム一覧 | `18x14 tiles` 前後 |
| モンスター一覧 | `18x14 tiles` 前後 |
| 配合親選択 | `18x12 tiles` 前後 |
| 図鑑 | `18x14 tiles` 前後 |

### 6.3 説明帯

- 下部固定 `2 tiles`
- 上のリストと罫線で分離
- アイテム / スキル / モンスター / 記録物 の全部で使い回す

## 7. コンポーネント規格

### 7.1 window frame

| 部品 | 数 |
|------|---:|
| corner | 4 |
| edge | 4 |
| fill | 1 |
| divider | 2 |

### 7.2 cursor

| 項目 | ルール |
|------|--------|
| サイズ | `8x8 px` |
| 既定形 | 三角 or 爪型 |
| アニメ | 2F |
| 周期 | `24〜30f` |

### 7.3 marker

| 種別 | 用途 |
|------|------|
| HP danger marker | 危険時のみ |
| status chip | 毒、眠り、封印など |
| sort marker | 図鑑や一覧 |
| favorite lock | 誤配合防止 |

---

## 8. モバイル入力規格

### 8.1 logical touch area

| 対象 | 内部最小サイズ |
|------|---------------:|
| D-pad 各方向 | `24x24 px` |
| A / B ボタン | `24x24 px` |
| START / SELECT | `20x16 px` |
| メニュー項目 | `8 px` 行高でも touch 上は `16 px` 扱い |

### 8.2 既定配置

- 左下: D-pad
- 右下: A / B
- 上端右: START
- 上端左 or 上端中央: SELECT / log

### 8.3 禁止

- 内部解像度前提の見た目だけ小さく、実タッチ領域も小さい実装
- battle 中にボタン位置が大きく跳ねること

## 11. QA Checklist

- 数値列が等幅で揃うか
- 説明帯 2 行で主要判断ができるか
- `160x144` のまま 1x で読めるか
- battle と menu で枠や余白の人格が変わっていないか
- touch 実装で内部 pixel 規格を壊していないか
