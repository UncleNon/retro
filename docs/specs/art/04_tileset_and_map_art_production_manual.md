## 1. Tile Fundamentals

### 1.1 Tile Size

| 項目 | 値 |
|------|----|
| 基本タイル | **8x8 px** 固定 |
| 内部解像度 | 160x144 px |
| 1画面 | 20x18 tiles |
| キャラクタースプライト | 16x16 px (2x2 tiles) |
| 大型マップオブジェクト | 16x16, 24x24, 32x32 px (倍数で構成) |

すべてのタイルは 8x8 px を最小単位とする。16x16 や 24x24 のオブジェクトも内部的には 8x8 の結合体として設計する。半端なサイズは禁止。

### 1.2 Tileset Organization

タイルセットシートは以下のレイアウトで統一する。

```
列幅: 16 tiles (128 px)
行数: 可変（最大64行 = 512 px）
シートサイズ上限: 128x512 px (1024 tiles)
```

**シート内の配置順序:**

| 行範囲 | 内容 |
|--------|------|
| Row 0-3 (0-31px) | 地面オートタイル (terrain autotile) |
| Row 4-7 (32-63px) | 水・液体オートタイル |
| Row 8-11 (64-95px) | 壁・障害物タイル |
| Row 12-15 (96-127px) | 建材・屋根タイル |
| Row 16-19 (128-159px) | ドア・窓・階段 |
| Row 20-27 (160-223px) | 装飾オブジェクト (家具、生活用品) |
| Row 28-35 (224-287px) | 植生・自然物 |
| Row 36-43 (288-351px) | 儀礼・記録・世界固有オブジェクト |
| Row 44-47 (352-383px) | 塔・門関連タイル |
| Row 48-55 (384-447px) | アニメーションタイル (水、炎、門脈動) |
| Row 56-63 (448-511px) | 予備・拡張領域 |

### 1.3 Tileset Naming Convention

```
TS-{scope}-{world_or_shared}-{variant}.png

scope:
  GND = ground/terrain
  BLD = building
  DEC = decoration
  DNG = dungeon
  TWR = tower/gate
  ANM = animation
  UI  = UI parts (別管理)

例:
  TS-GND-SHARED-base.png      共通地面タイル
  TS-GND-W001-pastoral.png    W-001 固有地面
  TS-BLD-SHARED-wood.png      共通木造建材
  TS-BLD-W007-stone_stair.png W-007 石段建材
  TS-DEC-W005-grave.png       W-005 墓苑装飾
  TS-TWR-SHARED-gate.png      共通門タイル
  TS-DNG-T05-ritual.png       T-05 儀式同心円ダンジョン
  TS-ANM-SHARED-water.png     共通水アニメーション
```

### 1.4 Maximum Tiles Per Tileset

| 制約 | 値 | 理由 |
|------|----|------|
| 1シートの上限 | 1024 tiles | 128x512 px シート |
| 1世界で参照するシート数上限 | 6 | VRAM と管理コストの均衡 |
| 1世界の実効タイル数 | 最大 256 推奨、上限 384 | パレット制約との両立 |
| 1マップで同時参照するシート数 | 4 以下推奨 | Godot TileMapLayer のパフォーマンス |

### 1.5 Shared vs World-Specific Tilesets

**共通タイルセット (SHARED):**

| シート | 内容 | 全世界で使用 |
|--------|------|:------------:|
| `TS-GND-SHARED-base` | 基本草地、土道、石畳、水 | Yes |
| `TS-BLD-SHARED-wood` | 木造壁、木製ドア、木窓、木屋根 | Yes |
| `TS-BLD-SHARED-stone` | 石壁、石ドア、石窓 | Yes |
| `TS-DEC-SHARED-life` | 井戸、樽、箱、看板、柵、ベンチ | Yes |
| `TS-TWR-SHARED-gate` | 門枠、門封印、境界杭、塔壁 | Yes |
| `TS-ANM-SHARED-water` | 水4F、松明4F、門脈動4F | Yes |

**世界固有タイルセット:**

各世界は共通シートに加え、1-3 枚の固有シートを持つ。固有シートには以下を含める。

- その世界でしか使わない地面バリアント
- 世界テーマの建材 (骨材、塩壁、鏡面石など)
- 世界固有の装飾オブジェクト (最低 8 種)
- 世界固有の儀礼・記録物 (最低 3 種)
- 門の世界固有パーツ

| COR-05 fracture-descent | W-014, W-017, W-020, W-021 | 鏡面床、残響壁、白泥地面 |

- 同じ草タイルを 3x3 以上並べない。必ず 2-3 種のバリアントを混在させる

- 枯草は世界の季節・気候に合わせて使用。温帯世界の中心部には置かない
- 湿草はwater タイルとの境界 1-2 タイルのみに使用

### 2.6 Water Tiles

水タイルはすべてアニメーション対象。

| ID | 名前 | フレーム数 | 速度 | ピクセル構成 |
|----|------|-----------|------|-------------|
| WTR-01 | 浅水 (静) | 4F | 400ms/F | 淡青ベタ + 白のハイライト点 1 個が 4F で位置移動 |
| WTR-02 | 浅水 (流) | 4F | 300ms/F | 淡青 + 白の横線 1px が右へ 1px ずつ移動 |
| WTR-03 | 深水 (静) | 4F | 500ms/F | 暗青ベタ + 淡青の揺らぎ点 2 個が交互に明滅 |
| WTR-04 | 深水 (流) | 4F | 350ms/F | 暗青 + 淡青の横線が右へ流れる |
| WTR-05 | 川 (細) | 4F | 300ms/F | WTR-02 と同様だが下方向へ流れる |
| WTR-06 | 滝表面 | 4F | 200ms/F | 白と淡青の縦線が下方向に高速移動 |
| WTR-07 | 波打ち際 | 4F | 500ms/F | 砂色→白→淡青→砂色の遷移。泡を 1px 白点で表現 |

**水タイルのアニメーションルール:**
- 全水タイルは 4 フレーム固定
- ハイライト点の移動は 1px/F 以下。激しい動きは禁止
- 深水と浅水は明度差で区別。深水は暗青 (彩度を落とす)、浅水は淡青
- 流れる方向はマップごとに統一。1 マップ内で流れの向きを混在させない
- 水面の白点は最大 2 個/タイル。多すぎるとちらつく

### 2.10 Autotile Rules: Terrain Edge Connection

本作は **47-tile フルオートタイル** を標準とする。

#### 47-tile System 概要

地形の接続は、各タイルの 8 方向 (上、下、左、右、左上、右上、左下、右下) の隣接情報に基づいて描画する。

```
47タイルの内訳:
- 中央 (全方向接続): 1
- 辺接続 (4辺の組み合わせ): 14
- 角接続 (内角): 4
- 辺+角の組み合わせ: 24
- 孤立: 1
- 端接続 (突端): 3
合計: 47
```

#### オートタイル対象の地形組み合わせ

| 地形A | 地形B | 優先度 | 境界表現 |
|-------|-------|--------|---------|
| 草 | 土 | 草が上 | 草が土に 1-2px かぶさる |
| 草 | 水 | 草が上 | 草の端 + 1px の暗色岸線 |
| 土 | 水 | 土が上 | 土の端 + 1px の暗色岸線 |
| 草 | 石畳 | 石畳が上 | 石畳の端が草を覆う |
| 草 | 砂 | 砂が上 | 砂が草に 1px 混じる (散点で遷移) |
| 雪 | 草 | 雪が上 | 雪の白点が草タイルに侵入 |
| 雪 | 氷 | 氷が上 | 氷のハイライトが雪に食い込む |
| 石畳 | 土 | 石畳が上 | 石畳の端が明確 |
| 沼 | 草 | 沼が下 | 草の端が茶緑に変色 |

#### 最小セットでの運用 (16-tile)

制作初期や低優先度の地形組み合わせでは 16-tile 簡易版を許可する。

```
16タイルの内訳:
- 4角 (外角): 4
- 4辺: 4
- 4内角: 4
- 中央 (全接続): 1
- 孤立: 1
- 横端2種: 2
合計: 16
```

**16-tile を許可する地形組み合わせ:**
- 沼 ↔ 草
- 氷 ↔ 雪
- 溶岩 ↔ 石
- 塩砂 ↔ 砂
- 世界固有の特殊地形同士

**47-tile を必須とする地形組み合わせ:**
- 草 ↔ 土 (全世界で最も多用)
- 草 ↔ 水
- 土 ↔ 水
- 石畳 ↔ 土
- 雪 ↔ 草

#### Corner Resolution Rules

角の処理は以下の優先順で解決する。

1. **辺が優先**: 辺の接続情報が角の接続情報より優先される
2. **内角は辺2本の交点にのみ発生**: 上と右が同じ地形なら右上内角が発生しうる
3. **外角は辺0本の頂点に発生**: 上も右も異なる地形なら右上外角
4. **孤立タイルは全辺が異なる地形の場合のみ**

#### Tiled Editor → Godot TileMapLayer マッピング

| Tiled 側 | Godot 側 |
|----------|----------|
| Terrain Set (type: corners + sides) | TileSet > TileSetAtlasSource > Terrain |
| 47-tile terrain | `terrain_mode = TERRAIN_MODE_CORNERS_AND_SIDES` |
| 16-tile terrain | `terrain_mode = TERRAIN_MODE_MATCH_SIDES` |
| Tile ID | atlas_coords (Vector2i) |
| Tile Property (collision) | Physics Layer の polygon |
| Tile Property (animation) | AnimatedTexture は使わず、TileSetAtlasSource の animation columns で定義 |

Tiled でのオートタイル設定は `Wang Tiles (Corner + Edge)` を使用し、エクスポート時に Godot の terrain 形式へ変換する。

## 3. Wall and Obstacle Tiles

### 3.1 Interior Walls

2D トップダウンにおける壁の高さ表現ルール。

| 壁種別 | 構成 | 高さ表現 |
|--------|------|---------|
| 標準内壁 | 8x8 の壁面タイル + 8x8 の壁上端タイル | 壁面は正面を描画 (レンガ/板/石)。壁上端は壁の天面を 2-3px の暗色帯で表現 |
| 高壁 | 8x8 壁面 x2段 + 8x8 壁上端 | 2段積み。上段を Layer 3 (overhead) に配置しプレイヤーの上に描画 |
| 半壁 / カウンター | 8x8 の壁面 1段のみ。上端を明るく | プレイヤーの半分の高さに見える。道具屋カウンター等 |

**内壁ピクセル構成:**

```
標準木壁 (8x8):
Row 0: 壁上端の影線 (最暗色 1px)
Row 1-2: 壁上端の天面 (暗茶)
Row 3-6: 壁面 (中茶ベタ + 板目の縦線 1px を 2 本)
Row 7: 壁下端の影 (最暗色 1px)

標準石壁 (8x8):
Row 0: 壁上端の影線 (最暗色 1px)
Row 1-2: 壁上端の天面 (暗灰)
```

### 3.3 Cliff Edges and Elevation Changes

崖は 2 タイル構成 (崖面 + 崖上端) で表現する。

```
崖上端タイル (8x8):
Row 0-3: 上の地面テクスチャ (草/土/雪)
Row 4: 崖端の影線 (最暗色 1px)
Row 5-7: 崖面の上部 (暗色の岩肌)

崖面タイル (8x8):
Row 0-6: 岩肌 (暗灰 + さらに暗い割れ目線 1-2 本)
Row 7: 崖下端の影 or 地面との接地線
```

**崖の方向バリエーション (各方向で 3 タイル):**
- 北向き崖 (上から見下ろす): 崖面タイル + 崖上端
- 東向き崖: 左辺が崖面
- 西向き崖: 右辺が崖面
- 崖角 (外角 4種、内角 4種)

**ジャンプダウン崖:** 特定の低い崖 (1段) はプレイヤーが上から下へ飛び降り可能。崖上端タイルの Row 4 を暗色ではなく中間色にして「越えられそう」な印象を与える。

- 色を使わない (白、灰、黒のみ。最大 6 色)
- 生きた植物を一切置かない
- 生活オブジェクトを一切置かない

**必須要素:**
- 白が画面の 80% 以上
- オブジェクトの形が不明確 (何であるか断定できない)

## 7. Tower and Gate Tile Design

### 7.1 Tower Exterior Tiles (Field Map)

塔は各世界に 1 基。フィールドマップ上では大型のマルチタイルオブジェクトとして描画する。

**基本寸法 (W-001 開始村の塔を基準):**

| 項目 | 値 |
|------|----|
| 可視外観幅 | 18 tiles (144 px) |
| 可視外観高 | 28 tiles (224 px) |
| 入口幅 | 4 tiles (32 px) |
| 前庭奥行き | 10 tiles (80 px) |

**塔外観の構成タイル:**

| パーツ | タイル数 | ピクセル構成 |
|--------|---------|-------------|
| 塔壁 (左) | 1x28 | 暗灰の石面。微妙な歪み (完全な直線にしない) |
| 塔壁 (右) | 1x28 | 左壁の非対称バリアント |
| 塔面 (中央) | 16x28 | 暗灰石 + 窓 (暗色穴) を不規則に。模様は反復するが周期がずれる |
| 塔頂部 | 18x4 | 尖塔ではなく、不完全に途切れた上端。「見切れている」印象 |
| 入口 | 4x3 | 暗色の開口部。門枠 TGO-02/03 で構成 |
| 前庭地面 | 18x10 | 通常地面から異物地面への遷移。GRS-01→GRS-06→DRT-07→STN-07 |

**塔の見た目ルール (Style Bible 準拠):**
- 完全左右対称を避ける
- 部分的な反復模様は許可
- 「宗教施設」「観測施設」「墓所」のどれとも断定できない混成感を保つ
- 塔壁の色温度は周辺より 1 段冷たい

### 7.2 Tower Interior Tiles

塔内部は通常の建物内部と異なる専用タイルを使用する。

| パーツ | ピクセル構成 | 特徴 |
|--------|-------------|------|
| 床 | 暗灰の石 + 微妙な模様線。FLR-07 (儀式模様床) を基調 | 同心円または螺旋の暗色線 |
| 壁 | 暗灰石 + 有機的な凹凸。通常石壁より歪んでいる | 壁面に骨色 (白灰) の筋が混じる |
| 天井影 | 上部 2 タイルに暗色のグラデーション帯 | 天井の高さを暗示 |
| 階段 | STR-04 (螺旋階段) を基本 | 塔内は螺旋で上下する |
| 窓 | WIN-03 (石窓) だが外の光が青白い | 通常の暖黄色ではない |
| 異常壁 | 壁面の一部が脈動する (4F アニメ)。暗色→異物色→暗色 | 門の影響が壁に出ている |

### 7.3 Gate Frame Tiles

門枠は「有機物と建材の中間」として設計する。

**門枠の基本構成 (16x24 = 2x3 タイル、左右一対):**

```
左枠:
[上部] 暗色の柱頭。角が丸く、歯や骨を連想させる凸部
[中部] 柱の胴。暗灰の石だが、表面に脈のような暗色の曲線
[下部] 台座。地面に食い込む。根のような暗色の線が地面タイルに浸食

右枠:
左枠と構成は同じだが、非対称。曲線の位置、凸部の数が異なる
```

**世界固有の門枠バリエーション:**

各世界の gate anatomy cue (05_world_catalog 参照) に合わせて門枠を変形する。

| world_id | 門名 | 枠の変形 |
|----------|------|---------|
| W-001 | 耳輪門 | 枠の上部に借り名札が連なる (小矩形の列) |
| W-002 | 胸郭門 | 枠が肋骨状に左右から合わさる |
| W-003 | 顎門 | 枠の上部に歯列状の凸部 |
| W-004 | 脊柱門 | 枠が脊椎のように節状に積み重なる |
| W-005 | 無孔墓門 | 枠に穴がない。のっぺりとした面 |
| W-006 | 湿喉門 | 枠が湿った質感。暗色の水滴点 |
| W-007 | 歯列石門 | 枠に規則的な歯列状の凸部 |
| W-008 | 囲い門 | 枠が円形に閉じかけている |
| W-009 | 白肋門 | 白い肋骨状の枠 |
| W-010 | 鰓門 | 枠に鰓のような横スリット |

### 7.4 Gate State Indicators

門は 4 つの状態を持ち、各状態でタイルの見た目が変わる。

| 状態 | 英名 | 門枠の変化 | 門面 (枠間) の変化 | アニメーション |
|------|------|-----------|-------------------|--------------|
| 封印 | Sealed | 枠の脈線なし。静止した暗灰 | TGO-04 門封印。暗色の膜 | なし |
| 傾聴 | Listening | 枠に微かな脈線 (暗色の 1px 線が現れる) | 膜に異物色の点が 1 個。微かに脈動 (4F, 800ms/F) | 遅い脈動 |
| 覚醒 | Awakened | 枠の脈線が明確。2-3 本 | 膜が薄まり、奥に暗い空間が見える。異物色点 2 個 | 中速脈動 (4F, 500ms/F) |
| 飢餓 | Hungry | 枠の脈線が激しく、枠自体が微動する | 膜がほぼ消え、奥の暗闇が見える。異物色が滲む | 速い脈動 (4F, 300ms/F) + 枠の微動 |
| 破裂 | Ruptured | 枠にひびが入る。破片タイルが周辺に飛散 | 完全に開放。奥が白い光 | 不規則な明滅 |

**異物色の定義:**
- パレットの「異物色」枠を使用 (Style Bible §4)
- 基本は冷紫 (`#6B4C8A` 付近) だが、世界によって微調整
- Hungry 以上で濁った金 (`#8A7B4C` 付近) が混じる
- Ruptured では反射白 (`#E8E0D0` 付近)

### 7.5 Transition Tiles: Normal Terrain → Tower-Influenced Terrain

塔の影響圏に入ると、地面タイルが段階的に変質する。

| 距離 (塔からのタイル数) | 地面の変化 | 効果 |
|------------------------|-----------|------|
| 20+ | 通常地面 (世界固有) | なし |
| 15-19 | 世界固有地面だが散点パターンに灰色点 1 個追加 | 色温度 -0.5 |
| 10-14 | 地面の暗色が増加。GRS-06 (枯草) 混在 | 色温度 -1.0 |
| 5-9 | DRT-07 (灰土) 主体。草がほぼ消える | 色温度 -1.5 |
| 1-4 | STN-07 (白石) 混在。地面にひび (TGO-05) | 色温度 -2.0 |
| 0 (塔直下) | 全面 STN-07 + TGO-05 (門傷跡) | 色温度 -2.5 |

この遷移は **グラデーション的** に行う。1 タイルごとの明確な線引きではなく、遷移帯を設けて自然に色温度を下げる。

## 9. Animation Tiles

### 9.1 Water Animation

| 対象 | フレーム数 | 速度 (ms/F) | 合計周期 | 動きの内容 |
|------|-----------|------------|---------|-----------|
| WTR-01 浅水静 | 4 | 400 | 1600ms | 白ハイライト点が 4 位置を巡回 |
| WTR-02 浅水流 | 4 | 300 | 1200ms | 白横線が右へ 1px/F 移動 |
| WTR-03 深水静 | 4 | 500 | 2000ms | 淡青点 2 個が交互明滅 |
| WTR-04 深水流 | 4 | 350 | 1400ms | 淡青横線が右へ移動 |
| WTR-05 川 | 4 | 300 | 1200ms | 淡青横線が下へ移動 |
| WTR-06 滝 | 4 | 200 | 800ms | 白+淡青縦線が下へ高速移動 |
| WTR-07 波打ち際 | 4 | 500 | 2000ms | 砂色→白→淡青→砂色の遷移 |

**水アニメーションの絶対ルール:**

### 9.2 Torch / Fire Animation

| 対象 | フレーム数 | 速度 (ms/F) | 動きの内容 |
|------|-----------|------------|-----------|
| LIT-01 松明 | 4 | 250 | 火の形が 3 パターンで変化 + 1F の明滅 |
| LIT-04 焚き火 | 4 | 300 | 火の形が 4 パターンで変化。サイズも微変 |
| LIT-05 蝋燭 | 4 | 400 | 黄点の位置が左右に 1px 揺れる |
| FRN-09 暖炉 | 4 | 300 | LIT-04 と同じだが小さめ |
| CHM-02 煙 | 4 | 500 | 灰白点が上へ 1px/F 移動して消滅 |

**火アニメーションの絶対ルール:**
- 火の色は 橙 → 黄 → 赤橙 → 橙 の 3 色以内で遷移
- 火の形は 2x2 px を超えない (松明)、3x3 px を超えない (焚き火)
- 隣接する複数の松明は **非同期** にする (1-2F ずらす) ことで自然感を出す

### 9.3 Gate Pulsation

| 状態 | フレーム数 | 速度 (ms/F) | 動きの内容 |
|------|-----------|------------|-----------|
| Sealed | 0 | - | 静止 |
| Listening | 4 | 800 | 異物色点 1 個が明度 ±10% で明滅 |
| Awakened | 4 | 500 | 異物色点 2 個が交互に明滅 + 枠の脈線が 1px 伸縮 |
| Hungry | 4 | 300 | 異物色が滲む (点から 2px の広がりへ) + 枠が 1px 横に微動 |
| Ruptured | 4 | 200 | 不規則明滅 (フレーム順が不定) + 白い光が溢れる |

**門アニメーションの絶対ルール:**
- 門の脈動は周囲のタイルアニメーションと **異なる周期** にする
- Hungry 以上では門周辺の地面タイル (TGO-05) も連動して微かに明滅
- 脈動の色は世界パレットの異物色を使用

|------|-----------|------------|-----------|
| GRS-05 長草 (風) | 2 | 600 | 草筋が右に 1px 傾斜 → 元に戻る |
| VEG-11 葦 | 2 | 700 | 上端が右に 1px → 元に戻る |
| TRE-01 広葉樹 (葉冠) | 2 | 800 | 明緑ハイライト点の位置が 1px 移動 |

**植生アニメーションの対象とする/しないの基準:**
- する: 長草、葦、大きな木の葉冠
- しない: 短草、花、低木、茸、根、蔦
- アニメーションは **2F** で十分。4F にしない (静かな揺れ)

47-tile のシート上配置は以下の 6x8 グリッド (48 セル、1 セル未使用) に統一する。

この配置は Tiled の Wang Tile セットと 1:1 で対応する。Godot へのインポート時に `terrain_mode = TERRAIN_MODE_CORNERS_AND_SIDES` で読み込む。

### 10.3 16-tile テンプレート配置

```
Row 0: [外角TL] [辺T]   [外角TR] [孤立]
Row 1: [辺L]    [中央]  [辺R]    [突端T]

```

### 10.4 Edge Cases

#### 地形が水と会う場合
- 水の岸辺は常に 1px の暗色線 (岸線) で区切る
- 草→水の遷移では、草の最下部 2px が暗緑に変色 (湿り表現)
- 砂→水の遷移では、砂の最下部 2px が暗色に (湿り砂)
- 石畳→水は岸線のみ。石畳自体は変色しない

#### 草が道と会う場合
- 草→土道の遷移では、草が土の上に 1-2px かぶさる
- 道の端に草が侵入する形。道側が削られることはない
- 石畳→草は石畳の端が明確。草が石畳に侵入しない

#### 3 地形が交わる三叉点
- 三叉点では最も優先度の高い地形の角タイルを使用
- 優先度: 水 > 石畳 > 土 > 草 > 砂 > 雪
- 三叉点タイルは専用に作らない。2 地形の角タイルの組み合わせで対応

#### 壁と地面の境界
- 壁は autotile ではなく手配置
- 壁の下端と地面の接触は壁タイルの Row 7 (影線) で処理
- 壁タイルは地面タイルの上に Layer 2 として重ねる

### 10.5 Tiled Editor → Godot TileMapLayer 詳細マッピング

| 工程 | 手順 |
|------|------|
| 1. Tiled でタイルセット作成 | `.tsx` ファイル。画像参照 + Wang Tile 設定 |
| 2. Wang Tile の color 設定 | 各地形に color を割り当て。corner + edge 両方設定 |
| 3. マップ描画 | Terrain ブラシで塗る。autotile が自動適用される |
| 4. エクスポート | `.tmx` (XML) または `.tmj` (JSON) で保存 |
| 5. Godot インポート | `gdscript` のカスタムインポーター or `Tiled Importer` addon |
| 6. TileSet 設定 | `TileSetAtlasSource` に画像を登録。Terrain を設定 |
| 7. TileMapLayer 配置 | Layer 0-4 に対応する TileMapLayer ノードを作成 |

**注意事項:**
- Tiled の Wang Tile ID と Godot の terrain bit mask は直接対応しない場合がある。変換テーブルを用意する
- Tiled のオブジェクトレイヤーは Godot の `Node2D` + `Area2D` に変換する
- collision shape は TileSet の Physics Layer で定義し、マップ側では設定しない

## 11. Map Layer Structure

### 11.1 Layer 定義

| Layer | 名前 | Godot ノード名 | Z-Index | 内容 |
|-------|------|----------------|--------:|------|
| 0 | Ground | `TileMapLayer_Ground` | 0 | 歩行可能な基本地面。草、土、石畳、砂、雪、水 |
| 1 | GroundDecor | `TileMapLayer_GroundDecor` | 1 | 非衝突の地面装飾。花、ひび、落ち葉、血痕、模様 |
| 2 | Walls | `TileMapLayer_Walls` | 2 | 衝突ありの壁、障害物、家具。Physics Layer 有効 |
| 3 | Overhead | `TileMapLayer_Overhead` | 10 | プレイヤーの上に描画。屋根、天蓋、橋の手すり |
| 4 | Events | `TileMapLayer_Events` | -1 | 不可視。イベントトリガー、エンカウントゾーン、ワープ |

### 11.2 各 Layer の詳細ルール

#### Layer 0: Ground
- すべてのタイルを隙間なく埋める。空きタイルは禁止
- autotile 対象の地形はすべてこの Layer に配置
- 水タイル (WTR-*) もこの Layer。アニメーション付き
- 室内の床 (FLR-*) もこの Layer

#### Layer 1: GroundDecor
- collision なし。プレイヤーは上を歩ける
- 配置するもの: VEG-01〜04 (花), VEG-12 (枯葉), ROK-01 (小石), TGO-05 (門傷跡)
- DNG-T01-03 (不穏な床跡), DNG-T04-01 (散乱文書) もここ
- 地面の色味を微調整する「フィルターレイヤー」としても使用
- 塔遷移タイル (§7.5) の灰色点追加もこの Layer で重ねる

#### Layer 2: Walls
- collision あり。`Physics Layer 0` に衝突ポリゴンを設定
- 配置するもの: 壁タイル (IWL-*, EWL-*), 岩 (ROK-02〜06), 木の幹 (TRE-* 下半分), 家具 (FRN-*), オブジェクト (VOB-*, RIT-*, REC-*)
- 半壁 (カウンター) は衝突ありだが高さが低い。`Physics Layer 0` に通常の矩形ポリゴン

#### Layer 3: Overhead
- collision なし。描画のみ
- Z-Index を 10 に設定し、プレイヤー (Z-Index 5) の上に描画
- 配置するもの: 屋根 (ROF-*), 木の葉冠 (TRE-* 上半分), 天蓋 (TRE-06), 橋の手すり上部, 壁の上端
- 半透明処理: プレイヤーが overhead タイルの下にいるとき、overhead タイルの alpha を 0.5 に下げる (実装は shader で対応)

#### Layer 4: Events
- 完全不可視。Runtime で visible = false
- Tiled ではオブジェクトレイヤーとして管理
- 配置するもの:

| オブジェクト種別 | プロパティ |
|-----------------|-----------|
| `npc_spawn` | `npc_id`, `facing`, `movement_pattern` |
| `inspect_point` | `inspect_id`, `text_key`, `item_grant` |
| `event_trigger` | `event_id`, `trigger_type` (step/interact), `one_shot` |
| `encounter_zone` | `table_id`, `min_steps`, `max_steps`, `terrain_rate` |
| `warp_point` | `target_map`, `target_x`, `target_y`, `facing` |
| `door_trigger` | `target_map`, `door_state` (open/closed/locked) |

### 11.3 Z-Ordering Rules

| 要素 | Z-Index | 備考 |
|------|--------:|------|
| Layer 0 (Ground) | 0 | 最背面 |
| Layer 1 (GroundDecor) | 1 | 地面の上 |
| Layer 2 下部 (壁の下半分) | 2 | |
| NPC スプライト | 3-5 | Y 座標に応じて動的に変化 |
| プレイヤースプライト | 5 | Y 座標に応じて動的に変化 |
| Layer 2 上部 (壁の上半分) | 6-8 | 壁の上端。プレイヤーの後ろ |
| Layer 3 (Overhead) | 10 | 最前面。屋根、天蓋 |

**Y-Sort ルール:**
- プレイヤーと NPC は `y_sort_enabled = true` の親ノード下に配置
- Layer 2 の壁タイルは、壁の下端 Y 座標でソートする
- プレイヤーの Y 座標が壁の下端より上なら、プレイヤーは壁の前に描画される
- プレイヤーの Y 座標が壁の下端より下なら、プレイヤーは壁の後ろに描画される

**実装方法:**
- Godot の TileMapLayer は Y-Sort を直接サポートしない
- 壁の上半分 (見えている面) を Layer 2 に、壁の上端を Layer 3 に分けることで擬似的に対応
- 大型オブジェクト (井戸、クレーン等) は TileMapLayer ではなく独立 Sprite2D として配置し、Y-Sort に参加させる

### 12.2 Partial Blocking (One-Way Ledges)

ジャンプダウン崖は上から下へのみ通過可能。

| タイル | 方向 | 実装 |
|--------|------|------|
| 崖上端 (南向き) | 上→下のみ | Physics Layer 3。`one_way_collision = true`, `one_way_direction = Vector2(0, 1)` |

| 崖上端 (西向き) | 右→左のみ | `one_way_direction = Vector2(-1, 0)` |

**ジャンプダウンの演出:**
- プレイヤーが崖端タイルに向かって歩くと、自動的に 2 タイル分飛び降りる
- 飛び降り中は入力無効 (0.3 秒)
- 着地時に 1F のスクワッシュ (座標は変えずスプライトを 1px 下にずらして戻す)

塔の影響圏に入ると、パレットを段階的にシフトする。

| 距離帯 | パレット操作 |
|--------|------------|
| 通常 | 世界固有パレットそのまま |
| 15-19 tiles | 最も暖かい色を 1段灰寄りに |
| 10-14 tiles | 暖色系を 2段灰寄りに。Row 2 の色を 1色追加 |
| 5-9 tiles | Row 1 の色を半減し、Row 2 + Row 0 を主体に |
| 0-4 tiles | Row 0 + Row 2 暗部 + 白のみ。暖色を完全に排除 |

**実装方法:**
- パレットシフトはタイルを差し替えることで実現 (runtime shader ではなく、遷移帯ごとに別タイルを用意する)
- 1世界あたり遷移タイルは 20-30 種で十分 (基本地面の灰バリアント + 枯草 + 灰土)

### 13.4 Indoor vs Outdoor Palette Adjustments

| 環境 | パレット調整 |
|------|------------|
| 屋外 (昼) | 世界固有パレットそのまま |
| 屋内 (明) | 全体を 1段暗く。窓から光が入る位置のみ明るい色を残す |
| 屋内 (暗) | 全体を 2段暗く。松明/蝋燭の周辺 2 タイルのみ暖色を残す |
| 洞窟 | Row 0 + Row 2 暗部のみ。光源周辺のみ暖色点 |
| 塔内部 | Row 0 + Row 2 暗部 + 異物色。暖色完全排除 |

**実装注意:**
- 「暗い」は黒に近づけるのではなく、**彩度を落とし明度を下げる**
- GBC 的な表現では、暗い場所は色数を減らすことで表現する (4色まで)
- 光源周辺のみ色数を増やすことで明暗のコントラストを出す

### 13.5 Night / Dusk Palette Shifts

本作では昼夜変化は実装しない (GBC 制約の模倣 + 開発工数)。ただし、ストーリー進行で特定のマップが「夕暮れ」「夜」になるイベントを持つ場合、以下のルールで対応する。

| 時間帯 | パレット操作 |
|--------|------------|
| 夕暮れ | Row 1 の色を橙寄りに 1段シフト。Row 2 を暗く |
| 夜 | 全色を 2段暗く + 青寄りにシフト。Row 3 の白を青白に |
| 月明かり | 夜パレット + 1色だけ明るい青白を追加 (月光表現) |

これはイベント専用の差し替えパレットとして用意し、通常ゲームプレイでは使用しない。

## 14. AI Generation Pipeline for Tilesets

### 14.1 Prompt Structure

AI 画像生成ツール (Stable Diffusion, Midjourney 等) でタイルセットを生成する際のプロンプト構造。

**基本プロンプトテンプレート:**

```
[STYLE] GBC-era pixel art tileset, 8x8 pixel grid, limited palette,
{palette_colors} colors maximum, clean pixel edges, no anti-aliasing,
no sub-pixel rendering, 1px black outline where applicable

[CONTENT] {tileset_type} tiles for {world_description},
{specific_tiles_list}

[LAYOUT] arranged in a {columns}x{rows} grid, each tile exactly 8x8 pixels,
no gaps between tiles, transparent background

[MOOD] {world_mood_keywords}

[NEGATIVE] blurry, anti-aliased, gradient, realistic, photograph,
high resolution detail, smooth edges, glow effects, lens flare,
modern UI elements, 3D rendering
```

**例: W-001 地面タイル:**

```
[STYLE] GBC-era pixel art tileset, 8x8 pixel grid, limited palette,
12 colors maximum, clean pixel edges, no anti-aliasing,
no sub-pixel rendering, 1px outline in darkest color

[CONTENT] ground/terrain tiles for a pastoral meadow village,
short grass (3 variants), tall grass (2 variants), dirt path (3 variants),
dried grass (2 variants), grass-to-dirt autotile edges (16 tiles)

[LAYOUT] arranged in a 16x4 grid, each tile exactly 8x8 pixels,
no gaps between tiles, transparent background

[MOOD] pastoral, worn, earthy, overcast warmth, faded straw, quiet unease

[NEGATIVE] blurry, anti-aliased, gradient, realistic, photograph,
bright saturated colors, cheerful, cute, modern
```

### 14.2 Consistent Autotile Edges の依頼方法

AI に autotile の整合的な端を生成させるのは困難。以下のワークフローを推奨。

1. **中央タイル + 基本4辺を AI 生成**: 5 タイルを一度に生成
2. **角タイルは手動作成**: AI 生成の 5 タイルを参考に、Aseprite で角タイルを手描き
3. **47-tile の残りは手動**: 基本 5 タイル + 角 4 タイルができたら、残りの組み合わせは手動でピクセル操作
4. **一貫性チェック**: すべての組み合わせを並べて端の接続を確認

**AI に直接 autotile セットを頼まない理由:**
- 8x8 のタイルが正確にグリッドに乗らない
- 端のピクセルが隣接タイルと一致しない
- 角の処理が不正確

### 14.3 Aseprite Cleanup Workflow

1. **AI 生成画像を Aseprite に読み込む**
2. **グリッド設定**: 8x8 px のグリッドを表示
3. **パレット制限**: 世界パレットの色のみに制限。`Sprite > Color Mode > Indexed` に変換し、パレットを手動設定
4. **タイル分割**: グリッドに沿って各タイルを確認
5. **修正項目 (必須)**:
   - サブピクセルの除去 (アンチエイリアスの残骸)
   - パレット外の色を最も近いパレット色に置換
   - アウトラインの修正 (1px 固定)
   - 端のピクセル連続性の確認 (隣接タイルと繋がるか)
6. **接続テスト**: 3x3, 5x5 のテストグリッドにタイルを並べて目視確認
7. **アニメーションフレーム作成**: 水や火のアニメーションフレームは手動で 4F を描く
8. **エクスポート**: PNG, indexed color, 透明背景

### 14.4 Common AI Artifacts to Fix

| 問題 | 頻度 | 修正方法 |
|------|------|---------|
| アンチエイリアスの残り | 非常に高い | indexed color 変換で自動除去 → 手動で確認 |
| グリッドずれ | 高い | 1px 単位でずれている。手動でシフト |
| 端のピクセル不一致 | 高い | 隣接タイルと並べて、接触するピクセルの色を手動で揃える |
| パレット逸脱 | 高い | indexed color 変換で強制。色味がずれた場合は手動で調整 |
| 光源の不統一 | 中 | AI が光源方向をタイルごとに変える。全タイルで「左上光源」に統一 |
| 情報量の過多 | 中 | 8x8 に詰め込みすぎ。散点を減らし、並びで読ませるように |
| 対称性の過剰 | 低 | 完全対称なパターンを崩す (1px のずらし) |
| 奥行き表現の混入 | 低 | パースがかかった表現の除去。フラットなトップダウンに |

## 15. Quality Checklist for Tilesets

### 15.1 Technical Checks

- [ ] 全タイルが正確に 8x8 px であること
- [ ] タイルシートのサイズが 128 px 幅 (16 タイル) であること
- [ ] 透明背景が使用されていること (地面タイルを除く)
- [ ] PNG 形式、indexed color であること
- [ ] ファイル名が命名規則 `TS-{scope}-{world}-{variant}.png` に従っていること
- [ ] 1 シートあたり 1024 タイル以下であること

### 15.2 Edge Continuity Checks

- [ ] 全 autotile の接続が正しいこと (3x3 テストグリッドで確認)
- [ ] 地面タイルの端ピクセルが隣接タイルと色が繋がること
- [ ] 壁タイルの上下の接続が自然であること
- [ ] 屋根タイルの 9 パーツが正しく組み合わさること
- [ ] 崖タイルの上端と崖面の接続が正しいこと

### 15.3 Autotile Connection Checks

- [ ] 47-tile セットの全 47 パターンが存在すること (必須地形)
- [ ] 16-tile セットの全 16 パターンが存在すること (準必須地形)
- [ ] 三叉点での表示が破綻しないこと
- [ ] 孤立タイル (周囲が全て異なる地形) が正しく表示されること
- [ ] Tiled 上での autotile ブラシが期待通りに動作すること
- [ ] Godot インポート後に terrain 設定が正しく反映されること

### 15.4 Palette Compliance Checks

- [ ] 使用色が世界パレットの範囲内であること
- [ ] 1 タイルセットの色数が 8-16 色以内であること
- [ ] 最暗色が外枠・影に使用されていること
- [ ] 異物色が門/塔関連タイルにのみ使用されていること
- [ ] 隣接世界とパレットが十分に差別化されていること
- [ ] 塔遷移帯のパレットシフトが段階的であること

### 15.5 World Identity Checks

- [ ] そのタイルセットだけ見て、どの世界かわかること
- [ ] 世界の禁忌/歪みがタイルデザインに反映されていること
- [ ] 固有装飾オブジェクトが最低 3 種含まれていること
- [ ] 色温度が世界の雰囲気と一致していること
- [ ] 同じ回廊 (COR-*) の他世界と、共有要素と差別化要素が明確であること
- [ ] Style Bible のビジュアル思想 (牧歌 + 冷たい異物 + 静かな禁忌) が感じられること
- [ ] 「完全な無垢さ」になっていないこと (必ず不穏さを含む)

### 15.6 Scale Consistency Checks

- [ ] ドアの高さが 16 px (キャラクター身長) と釣り合っていること
- [ ] テーブルの高さがキャラクターの腰程度 (8 px) に見えること
- [ ] 木のサイズがキャラクターの 1-2 倍の高さに見えること
- [ ] 井戸のサイズがキャラクターの横幅の 2 倍程度であること
- [ ] 階段の段差が歩行アニメーションと矛盾しないこと
- [ ] 建物の外寸と内装マップの広さが大きく矛盾しないこと (多少の拡大は RPG 慣例として許容)
- [ ] 1 タイル (8x8) が概ね人間の半歩〜一歩分の距離に見えること

### 15.7 Final Sign-Off Checklist (リリース前)

- [ ] 全 21 世界のタイルセットが揃っていること
- [ ] 共通タイルセット 6 シートが完成していること
- [ ] 全 autotile が Tiled + Godot で動作確認済みであること
- [ ] 全アニメーションタイルが正しい速度で再生されること
- [ ] 全 collision 設定が正しいこと (壁を通り抜けない、ダメージ床が機能する)
- [ ] 全 Layer の Z-Index が正しいこと (描画順の破綻がない)
- [ ] 塔遷移帯が 21 世界すべてで正しく機能すること
- [ ] 門の 5 状態がすべて視覚的に区別できること
- [ ] パレットが全世界で一貫していること (マスターパレットから逸脱していない)
- [ ] 1x 表示 (160x144) で全タイルが視認可能であること
