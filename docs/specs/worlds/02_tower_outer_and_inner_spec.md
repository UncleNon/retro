## 1. 目的

- 塔の外観と最初の内部導線を、**開始村から最初の門の向こう側へ越境するための実装仕様**として固定する
- ただのダンジョン入口ではなく、生活圏の延長にある異物として見せる
- 説明より先に違和感を積み、`生きた門` の存在を明かしつつも、用途や起源は言い切らない

---

## 2. 設計方針

- 外観は `聖堂 / 修道院 / 観測塔 / 墓所` のどれとも断定できない
- 近づくほど建築の寸法が人間基準からずれていく
- 最初の塔内部では通常エンカウントを発生させず、**環境そのものが敵対的に見える** 方向で緊張を作る
- 触れた物すべてが直接答えを出すのではなく、`生活物の延長なのに意味が違う` というズレを残す

## 5. 前庭トリガ

| トリガID | 座標 / 範囲 | 条件 | 効果 |
|----------|-------------|------|------|
| `TRG-TWR-001` | `forecourt-lower` | 初回侵入 | カメラを上へ 4 タイル寄せる |
| `TRG-TWR-002` | `forecourt-middle` | 初回侵入 | 村 ambient の鳥声と家畜音をミュート |
| `TRG-TWR-003` | `threshold-stone` | `EVT_VIL_010_TAG_TRACE_FOUND` | 主人公が足を止め、札の寸法との一致を視線で示す |
| `TRG-TWR-004` | `door-shadow` | 初回 | 外扉の隙間が自発的に開いて見える |

### 5.1 進入条件

- 初回塔進入に必須なのは `EVT_VIL_010_TAG_TRACE_FOUND`
- これはプレイヤーの移動を厳しく縛るためではなく、**札と門の寸法一致** を最低 1 度体験させるため
- `EVT_VIL_020_EMPTY_GRAVE_SEEN` と `EVT_VIL_030_WARNING_STAKE_SEEN` は任意。見ていると一部の内側メッセージが増える

## 7. 塔内部シーケンス

### 7.1 `MAP-TWR-001` 入口前室

| 項目 | 値 |
|------|----|
| サイズ | `32 x 24 tiles` |
| 入場座標 | `(16,21)` |
| 出口座標 | 北 `x 14-17, y 2` |
| エンカウント | なし |

目的:

- 村の外と内で `空気の密度が変わる` ことをプレイヤーに意識させる
- ここではまだ門を見せない

オブジェクト:

| オブジェクト | 座標 | 調べた時の役割 |
|--------------|------|----------------|
| 干からびた水盤 | `(10,10)` | 家畜の水桶に似ているが内側に家印 |
| 両側の灯架 | `(8,6)`, `(23,6)` | 火を焚いた跡がない |
| 扉脇の小溝 | `(13,18)`, `(18,18)` | 札が差し込めそうな幅 |

演出:

- 入場 0.8 秒は入力を受けるが、主人公が自発的に 1 歩ゆっくり進む
- カメラは中央固定。揺らさない

### 7.2 `MAP-TWR-002` 札廊 / 帳架回廊

| 項目 | 値 |
|------|----|
| サイズ | `40 x 24 tiles` |
| 入場座標 | `(20,21)` |
| 出口座標 | 北西 `x 8-11, y 2` |
| エンカウント | なし |

構造:

- 東西に長い廊下
- 壁一面に `札`, `薄い板`, `背表紙のない帳面` が並ぶ
- 生活道具の保管庫にも宗教施設の納骨棚にも見える

主要調査点:

| 調査点 | 座標 | 内容 |
|--------|------|------|
| 棚A | `(9,8)` | 札の寸法が村の家畜札と同じ |
| 棚B | `(18,7)` | 名前らしきものが削られている |
| 棚C | `(28,9)` | 家印だけ残っている |
| 床溝 | `(20,14)` | 札を差し込む向きが村の記録小屋と逆 |

トリガ:

| トリガID | 条件 | 効果 |
|----------|------|------|
| `TRG-TWR-101` | 棚A 調査 | `FLAG_TWR_TAG_SCALE_CONFIRMED` |
| `TRG-TWR-102` | 棚B 調査 | `FLAG_TWR_NAME_ERASURE_SEEN` |
| `TRG-TWR-103` | 棚C 調査 | `FLAG_TWR_HOUSEMARK_ONLY_SEEN` |

### 7.3 `MAP-TWR-003` 空洞階段室

| 項目 | 値 |
|------|----|
| サイズ | `24 x 40 tiles` |
| 入場座標 | `(10,35)` |
| 出口座標 | 北 `x 10-13, y 3` |
| エンカウント | なし |

構造:

- 中央に螺旋ではない、ゆるく撚れた上り道
- 壁に窓はないが、外光とは違う冷たい明るさが漏れる

演出:

- 5 段進むごとに、歩数 SE を 1 フレームだけ遅らせる
- `P1` で空碑を見ている場合のみ、途中の壁に浅い墓碑形の凹みを見つけられる

### 7.4 `MAP-TWR-004` 最初の門の間

| 項目 | 値 |
|------|----|
| サイズ | `32 x 32 tiles` |
| 入場座標 | `(16,27)` |
| 門中心 | `(16,10)` |
| エンカウント | なし |

構造:

- 円形に近い空間
- 中央奥に `生きた門`
- 四隅に用途不明の支柱。鐘楼にも柵にも見える

生きた門の見た目:

- 石の門枠ではなく、縦長の裂け目を肋骨めいた支柱が囲っている
- 奥に風景は見えない。半透明の膜でも光でもなく、**濡れた暗さ** が張っている
- 一定間隔で呼吸のように縮む

## 8. 最初の門の開き方

### 8.1 基本条件

| 条件 | 必須 |
|------|------|
| `EVT_VIL_010_TAG_TRACE_FOUND` | 必須 |
| `FLAG_TWR_TAG_SCALE_CONFIRMED` | 必須 |
| `FLAG_TWR_NAME_ERASURE_SEEN` | 必須 |
| `FLAG_TWR_HOUSEMARK_ONLY_SEEN` | 必須 |

### 8.2 開門シーケンス

1. 主人公が門前 `x 15-17, y 15-17` に入る
2. カメラ固定
3. 主人公が持ち歩いている仕事道具の札束を触る仕草を 1 回入れる
4. 門枠脇の細溝が村の家畜札の寸法と同じであることが視覚的に分かる
5. 札を差し込む動作は UI 化せず、自動演出で行う
6. 門が開くのではなく、膜が `こちらを認識する` ように浅く波打つ

### 8.3 演出ルール

- 門は光らない。内部が濡れたように暗くうごめく
- SE は金属よりも喉、革、水に近い音を混ぜる
- 主人公名は表示しないが、村の方言に近い抑揚の囁きを 1 回だけ鳴らす

## 11. 実装メモ

- 初回塔内部は通常戦闘なし。緊張は環境で担保する
- `MAP-TWR-002` の棚は、すべて調べられる必要はない。3 つの必須棚 + 2 つのダミーで十分
- 生きた門は `ポータル面 + 呼吸アニメ + 枠の微振動` の 3 層で構成する
- 外扉は開閉アニメを大げさにしない。そもそも開いていたのか、認識した結果開いたのか曖昧にする
