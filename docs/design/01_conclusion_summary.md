# 1. 結論サマリー

> **Archived**: この文書は旧設計。現行の source of truth は `docs/requirements/`。

本案件は『Unity 6 LTS + Aseprite』を中核にし、AI画像生成は『Nano Banana 2 / Nano Banana Pro』と『GPT Image 1.5』を役割分担させる二段構えが最適である。再現すべきは単なる4色・低解像度ではなく、"狭い画面でも迷わない設計""短い周回ループ""強い収集圧""データの外部化による量産可能性"である。Nintendo公開情報ではオリジナルGame Boyは160 x 144画素・4階調で、Pan Docsは8x8/8x16スプライトや40スプライト/画面・10スプライト/走査線という制約を示している [R1][R2]。この制約が、シルエット優先・UI簡素化・地形再利用・高密度テキスト誘導という設計思想を生んだ。

また、Square Enixが『ドラゴンクエストモンスターズ テリーのワンダーランドRETRO』をスマートフォン向けに展開し、オリジナル準拠のピクセル表現・簡潔なシステム・UIカスタマイズを現代端末で成立させていることから、同種の表現が今でも十分に実用的であることは確認できる [R3]。一方で、同じ"雰囲気"を狙っても、固有キャラクター・地名・アイコン・メニュー枠・モンスター外形を寄せ過ぎると著作権・商標・トレードドレス上のリスクが上がるため、本企画は必ずオリジナルIPとして設計する。

> **本書で採用する最終推奨**
>
> エンジンはUnity 6 LTS（URP 2D + 2D Pixel Perfect）、最終ドット制作はAseprite、AI素材生成はNano Banana 2/Proで高速な案出し・多バリエーション生成、GPT Image 1.5で厳密な差分編集と参照画像ベースの一貫性維持を行う。Midjourneyはムードボード用途、Adobe Fireflyは企業案件のブランド整合用途、Recraftはベクター/UI補助用途として扱い、主制作基盤にはしない。

| **項目**       | **推奨**                                    | **理由**                                                                                     |
|----------------|---------------------------------------------|----------------------------------------------------------------------------------------------|
| ゲーム構造     | トップダウン探索 + 3体PTターン制 + 合成育成 | DQM級の育成中毒性と携帯機向け短いセッション性を両立しやすい。                                |
| 画面設計       | 内部解像度 320x288 / 可視20x18マス          | Game Boyの20x18タイル密度を維持しつつ、現代端末で文字とUIを読みやすくする。                  |
| エンジン       | Unity 6 LTS                                 | URP 2D + 2D Pixel Perfect、Tilemap、整数スケーリング、Android/iOS出力が揃う。                |
| 最終ドット制作 | Aseprite                                    | tilemap layer、linked cels、CLI exportで量産/修正/自動出力に強い [R7]。                      |
| AI案出し       | Nano Banana 2/Pro                           | 参照画像・複数案・subject consistency・高速反復に強い [R9]。                                 |
| AI差分編集     | GPT Image 1.5                               | 入力画像保持、参照画像生成、編集精度、反復編集の品質が高い [R8]。                            |
| 法務方針       | オリジナルIP前提                            | ゲームの方法や一般的なアイデア自体は保護されないが、具体的表現は保護対象になり得る [R13]。   |
